## Achados / CONCERNS

### CONCERN-1 — Canonical incorreto em `/academy/turmas`

`/academy/turmas` retorna `<link rel="canonical" href="https://joaoguirunas.com">` — aponta para a homepage, não para a própria página. Idem para OG tags (`og:title=João Guirunas` em vez de "Turmas disponíveis"). Afeta SEO da LP pública de turmas.

**Recomendação:** sites-dev-alpha ou sites-dev-beta corrigir metadata de `/academy/turmas` em iteração futura.

### CONCERN-2 — Cohort `curso-online-padrao` ausente da LP `/academy/turmas`

A LP pública lista apenas `Turma 1 — Mentoria Claude Code + AIOX` (slug `turma-1`, R$ 8.700,00, 12 vagas). O cohort `curso-online-padrao` que F8.7 testa **não aparece**. Possibilidades:
- Cohort não foi criado em prod ainda (esperado — João vai criar)
- Cohort criado mas sem `status` que permita listagem pública
- Lógica de listagem só lista cohorts com vagas restantes > 0

**Bloqueio:** AC4 não pode ser executado até cohort `curso-online-padrao` existir + aparecer na LP ou ser acessível via `/academy/checkout/curso-online-padrao` direto.

### CONCERN-3 — Headers de segurança incompletos

Resposta de `/` contém apenas `Strict-Transport-Security` e `Server: Vercel`. **Ausentes:**
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY` ou `SAMEORIGIN`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Content-Security-Policy`
- `Permissions-Policy`

**Impacto:** baixo para MVP; site é puro Next.js sem inputs sensíveis públicos. Mas **Best Practices Lighthouse** vai pontuar abaixo de 100 sem esses headers.

**Recomendação:** sites-devops adicionar `next.config.ts → headers()` em iteração de hardening pós-launch.

### CONCERN-4 — Cohort UUID hardcoded no payload SSR

`/academy/turmas` SSR contém `40000000-0000-0000-0000-000000000001` no payload — UUID determinístico padrão de seed. Se o seed foi rodado em prod, há risco de dados de demo. **Pendente confirmação:** João verificar se `seed.sql` foi rodado em prod (ver `ops/launch-readiness.md` que diz NÃO deve ser).
