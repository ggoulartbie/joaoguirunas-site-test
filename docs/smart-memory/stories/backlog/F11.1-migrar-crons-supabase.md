# Story F11.1: Migrar crons para Supabase pg_cron + Edge Functions

## Objetivo
Eliminar dependência do Vercel Pro para cron jobs, migrando toda a lógica agendada para Supabase (pg_cron + Edge Functions). Deploy fica 100% no plano Hobby.

## Acceptance Criteria

- [ ] AC1: Lógica de expirar matrículas (`expire_memberships`) migrada para pg_cron — SQL puro, roda 1x/dia
- [ ] AC2: Lógica de lembretes D-15/D-7/D-3 migrada para Edge Function chamada via pg_cron
- [ ] AC3: Lógica de renovação automática migrada para Edge Function chamada via pg_cron
- [ ] AC4: Lembrete "sessão em 1 hora" migrado para Edge Function chamada via pg_cron (schedule: `0 * * * *`)
- [ ] AC5: `/api/cron/daily` e `/api/cron/hourly` em Next.js removidos ou mantidos como fallback desactivado
- [ ] AC6: `vercel.json` sem nenhum cron job — zero dependência de plano Pro
- [ ] AC7: Edge Functions deployadas e activas no projecto Supabase `mksmmpfyqowuzjcchhkl`

## Notas
Pós-launch. Não bloqueante para go-live.
