# Story F3.4: Renderização triplo formato (MDX, HTML, Markdown)

## Objetivo

Renderizar o `lessons.content` conforme `lessons.content_format`, suportando MDX, HTML sanitizado e Markdown, com syntax highlighting e componentes customizados.

## Acceptance Criteria

- [ ] AC1: `lib/content/mdx.ts` exporta `renderMDX(content)` usando `next-mdx-remote/rsc`
- [ ] AC2: Componentes MDX customizados: `<Callout>`, `<CodeBlock>` (com botão copy), `<Tabs>`, `<TabPanel>` — todos com estilo do KV
- [ ] AC3: `lib/content/html.ts` exporta `renderHTML(content)` que usa `isomorphic-dompurify` server-side antes de inserir HTML; whitelist de tags/attrs definida
- [ ] AC4: `lib/content/markdown.ts` exporta `renderMarkdown(content)` usando `react-markdown` + `rehype-shiki` para syntax highlighting (tema dark + light)
- [ ] AC5: Componente `<LessonContent format="..." content="..." />` que faz switch e renderiza apropriadamente
- [ ] AC6: Imagens em MDX/Markdown servidas via `next/image` quando possível
- [ ] AC7: Estilos: tipografia consistente com Tailwind Typography (`prose`) ajustado pro tema do site
- [ ] AC8: Performance: HTML sanitizado em **server-side** (não client) para evitar XSS

## Escopo

**IN:**
- 3 renderers + componentes customizados MDX
- Sanitização HTML server-side
- Switch component
- Estilo prose alinhado ao KV

**OUT:**
- Editor admin de conteúdo (Fase 6)
- Imagens responsivas otimizadas (Fase 8 polimento)

## Contexto Técnico

- Conteúdo de admin é confiado mas **HTML deve sempre passar por DOMPurify** (D2.13 do ADR-001)
- Shiki em RSC: usar versão server-only, evitar bundle no client
- MDX: componentes customizados ficam em `components/mdx/`
- Risco crítico (ADR-001): HTML XSS — sanitização **antes** de salvar OU **antes** de renderizar (escolha: ambos, defesa em profundidade)
