# Story F8.1: Notifications in-app + onboarding

## Objetivo

Sininho de notifications no header do aluno (lê tabela `notifications`) e wizard de onboarding no primeiro login.

## Acceptance Criteria

- [ ] AC1: Componente `<NotificationBell>` no header da área do aluno com badge de count não lidas
- [ ] AC2: Dropdown com últimas 10 notifications + link "Ver todas" (página `/notificacoes`)
- [ ] AC3: Click em notification marca como lida (`read_at=now()`) e navega pra `action_url`
- [ ] AC4: Notifications criadas em pontos: novo material em curso ativo, matrícula expirando 7 dias, certificate pronto, nova reply em thread que o user criou
- [ ] AC5: Onboarding wizard no primeiro login (detectado por `cohort_members.first_login=false` ou flag em profile):
  - Step 1: foto + bio
  - Step 2: tour rápido das áreas
  - Step 3: link pro group_url da primeira cohort
  - Skip permitido
- [ ] AC6: Realtime opcional via Supabase Realtime para badge atualizar sem reload (nice-to-have)

## Escopo

**IN:**
- Sininho + dropdown + página /notificacoes
- Triggers que criam notifications
- Wizard de onboarding

**OUT:**
- Push notifications mobile (post-MVP)
- Configuração granular de quais notifications receber (post-MVP)

## Contexto Técnico

- Adicionar coluna `profiles.onboarded_at timestamptz` para gating do wizard
- Realtime: `supabase.channel('notifications').on(...)` se decidirmos
- Tabela `notifications` já existe (F1.4)
