# Story 2.2: Skeleton loading no RevosForm

## Objetivo
Adicionar um estado de loading visível com `animate-pulse` dentro do container do `RevosForm` enquanto o script externo do Revos não renderiza o formulário. Em conexões lentas, o usuário vê uma área vazia por vários segundos ao chegar em `/mentoria#inscricao` — o skeleton elimina essa percepção de "página quebrada".

## Acceptance Criteria

- [ ] AC1: `src/app/mentoria/revos-form.tsx` adiciona estado `isLoading: boolean` (`useState(true)`).
- [ ] AC2: Enquanto `isLoading === true`, o container renderiza skeleton com 3 blocos `animate-pulse bg-white/[0.06] rounded h-10 mb-3` + um botão desabilitado "Carregando..." (ou placeholder equivalente).
- [ ] AC3: O skeleton é removido quando o script do Revos dispara o evento `load` (listener `script.addEventListener('load', () => setIsLoading(false))` antes do append).
- [ ] AC4: Fallback de segurança: `setTimeout(() => setIsLoading(false), 5000)` — remove o skeleton mesmo se o evento `load` nunca disparar (ex.: script bloqueado por ad-blocker). O timeout é limpo no cleanup do `useEffect`.
- [ ] AC5: Altura mínima do container permanece `min-h-[400px]` — skeleton preenche esse espaço com os blocos pulsantes.
- [ ] AC6: Skeleton fica dentro do mesmo `<div id={DIV_ID}>` (ou irmão) — se for irmão, o skeleton é condicional via `{isLoading && ...}` para não interferir na injeção do form pelo Revos.
- [ ] AC7: `pnpm build` conclui sem erros.
- [ ] AC8: Teste manual em `/mentoria#inscricao`: simular rede lenta (Chrome DevTools → Slow 3G) e validar que o skeleton aparece e é substituído pelo form.
- [ ] AC9: Teste manual com ad-blocker ativo ou script Revos bloqueado: skeleton desaparece após 5s (AC4).

## Escopo

**IN:**
- Modificar `src/app/mentoria/revos-form.tsx` adicionando `isLoading` state + skeleton + listener `load` + fallback timeout.
- Garantir que skeleton respeita design system (accent `#FF4400` se houver elementos coloridos; tons `white/[0.06]` para blocos neutros).
- Atualizar `docs/smart-memory/agents/ux/components.md` — §RevosForm adicionar nota sobre estado de loading implementado.

**OUT:**
- Mudar o form ID ou endpoint do Revos.
- Refatorar o componente para usar `next/script` (já foi decisão anterior ficar com `document.createElement` — commit `f6c55d9` / `2ca12f2`).
- Instrumentar analytics no evento de load (fora do escopo SEO/conversão desta story).
- Estilização do skeleton para simular 1:1 o form real (basta sinalização de loading).

## Notas técnicas

**Arquivo atual:** `src/app/mentoria/revos-form.tsx` (28 linhas). O `useEffect` cria `script` via `document.createElement`, anexa ao body, faz cleanup no unmount. O container é `<div id={DIV_ID} className="min-h-[400px] w-full max-w-full overflow-hidden" />`.

**Shape sugerido após fix:**

```tsx
export function RevosForm() {
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    document.querySelectorAll(`script[data-revos="${FORM_ID}"]`).forEach(el => el.remove());

    const script = document.createElement('script');
    script.src = SCRIPT_SRC;
    script.setAttribute('data-revos', FORM_ID);
    script.setAttribute('data-form-id', FORM_ID);
    script.async = true;
    script.addEventListener('load', () => setIsLoading(false));
    document.body.appendChild(script);

    const fallback = setTimeout(() => setIsLoading(false), 5000);

    return () => {
      script.remove();
      clearTimeout(fallback);
    };
  }, []);

  return (
    <div className="min-h-[400px] w-full max-w-full overflow-hidden">
      {isLoading && (
        <div className="space-y-3" aria-hidden="true">
          <div className="animate-pulse bg-white/[0.06] rounded h-10" />
          <div className="animate-pulse bg-white/[0.06] rounded h-10" />
          <div className="animate-pulse bg-white/[0.06] rounded h-10" />
          <div className="h-12 flex items-center justify-center border border-white/[0.08] bg-white/[0.03] text-white/40 text-sm">
            Carregando formulário...
          </div>
        </div>
      )}
      <div id={DIV_ID} />
    </div>
  );
}
```

Notas:
- `aria-hidden="true"` no bloco do skeleton porque é decorativo (loading state já é comunicado pelo texto "Carregando formulário...").
- `<div id={DIV_ID} />` fica fora do conditional para o Revos conseguir mutar o DOM mesmo antes do evento `load` disparar.

## Contexto Técnico

- **Script origem:** `https://revos.growthsales.ai/embed.js?form_id=a11d7cc4-17b8-400e-94e4-0f27ca47e9a4` (terceiro; não temos controle do payload).
- **Integração atual:** o componente foi ajustado 3 vezes recentemente (commits `bf525f9`, `f6c55d9`, `2ca12f2`) por problemas de SPA navigation — cuidado para não regredir esses fixes.
- **Browser support:** `script.addEventListener('load', ...)` é universal (IE9+). `setTimeout` idem.
- **Risco:** se o Revos mudar para carregar o form antes do `load` event (ex.: inline synchronous render), o skeleton some tarde demais. O fallback de 5s cobre esse caso. Aceita-se risco residual de 5s de skeleton a mais em edge cases extremos.
