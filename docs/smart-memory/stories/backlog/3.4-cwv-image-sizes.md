# Story 3.4: CWV — sizes em imagens responsivas

## Objetivo
Adicionar prop `sizes` nas imagens responsivas que estão sem ela, reduzindo o tamanho de download desnecessário em mobile (CWV-02 do seo-audit).

## Acceptance Criteria

- [ ] AC1: `src/app/mentoria/page.tsx` — cards de facilitadores recebem `sizes="(max-width: 640px) 176px, (max-width: 768px) 224px, 288px"`.
- [ ] AC2: `src/shared/components/ui/SkillPage.tsx` — bgImage fill recebe `sizes="100vw"`.
- [ ] AC3: `src/app/mentoria/page.tsx` — corrigir `SoftwareApplication.author.name` no `generateSkillJsonLd` para usar nome real em vez de handle.
- [ ] AC4: `tsc --noEmit` passa sem erros.

## Contexto

- `mentoria/page.tsx` facilitadores: `width={288} height={288}` com classes responsivas Tailwind — sem `sizes` o browser baixa 288px em mobile.
- `SkillPage.tsx` bgImage: `fill` sem `sizes` — browser assume 100vw, potencial download excessivo em mobile.
- `SoftwareApplication.author.name`: usa `author.replace('@', '')` gerando `joaoguirunas` em vez de `João Guirunas`.
