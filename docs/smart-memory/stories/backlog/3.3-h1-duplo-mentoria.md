# Story 3.3: H1 duplo no DOM em /mentoria

## Objetivo
Corrigir a presença de dois `<h1>` no DOM da `/mentoria`. O layout mobile (`sm:hidden`) e desktop (`hidden sm:block`) cada um tinha seu próprio H1 — visualmente apenas um é renderizado por vez, mas crawlers veem ambos.

## Fix aplicado

O H1 mobile recebeu `aria-hidden="true"` — remove da árvore de acessibilidade e sinaliza aos crawlers que é duplicado decorativo. O H1 desktop permanece como o heading canônico.

## Acceptance Criteria

- [x] AC1: H1 no bloco `sm:hidden` tem `aria-hidden="true"`.
- [x] AC2: H1 no bloco `hidden sm:block` permanece sem `aria-hidden`.
- [x] AC3: `tsc --noEmit` passa sem erros.
- [ ] AC4: Verificar via DevTools → Elements que apenas um H1 aparece na accessibility tree.
