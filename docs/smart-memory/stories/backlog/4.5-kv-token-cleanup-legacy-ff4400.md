# Story 4.5: KV Token Cleanup — uniformizar #FF4400 legados para var(--color-accent)

## Objetivo
Substituir todas as ocorrências hardcoded de `#FF4400` no código por `var(--color-accent)` (ou o token Tailwind equivalente), para que o refinamento de cor do KV (de #FF4400 para #FF3A0E) seja refletido automaticamente em toda a UI.

## Origem
Concern não-bloqueante levantado pelo QA no Epic 4 (2026-04-26): "coexistência de tokens #FF4400 (legacy) e #FF3A0E (novo accent)".

## Acceptance Criteria
- [ ] AC1: `grep -r "#FF4400" src/` retorna zero resultados (exceto comentários ou arquivos de documentação)
- [ ] AC2: Substituições feitas apenas onde o valor semântico é "accent/brand color" — não substituir onde a cor tem propósito específico diferente (ex: gradientes do símbolo GrowthWatermark)
- [ ] AC3: Build Next.js compila sem erros após substituição
- [ ] AC4: Verificação visual: nenhuma cor quebra visualmente (alguns CTAs podem ter ficado em #FF4400 hardcoded por opção deliberada)

## Escopo

**Arquivos prováveis:**
- `src/app/mentoria/page.tsx` — CTAs, badges de urgência
- `src/app/open-source/open-source-client.tsx` — card icons, badges
- Outros componentes em `src/shared/components/`

**OUT:**
- `src/shared/components/ui/growth-watermark.tsx` — o gradiente do diamante usa #FF4400/#FF1A00 intencionalmente (cor da marca, não do accent do site)
