# Story F9.16: Auditar components student críticos (VideoPlayer, Materials, Comments)

## Objetivo
Auditar internamente os components de maior risco em `src/components/student/*`: `VideoPlayer`, `MaterialsList`, `CommentsSection`, `LockedContent`, `MarkCompleteButton`, `NotificationBell`. Foco em XSS no render de conteúdo, leak de `video_id`, TTL de signed URLs, integridade de chamadas de Server Action.

## Acceptance Criteria
- [ ] AC1: `VideoPlayer.tsx`:
  - `videoId` só é renderizado quando `has_access === true` no Server Component (já validado em `aula/page.tsx:194`). Confirmar que NÃO é fetchado/passado como prop quando `hasAccess` for false.
  - Provider Vimeo/YouTube embed usa origem confiável (sem injetar URL arbitrária).
  - `saveProgress` chamado periodicamente — validar throttling para não floodar Server Action.
- [ ] AC2: `CommentsSection.tsx`:
  - Conteúdo do comment é renderizado escapado (sem `dangerouslySetInnerHTML` direto). Se há markdown → usa renderer com sanitização (DOMPurify ou similar).
  - Tentar injetar `<script>`, `<img onerror>`, `javascript:` em comment → renderiza como texto, não executa.
- [ ] AC3: `MaterialsList.tsx`:
  - Download via signed URL — TTL ≤ 1 hora (recomendado).
  - Signed URL gerada server-side (via Server Action), nunca exposta no JSON do Server Component sem expiração.
  - Acesso negado para STUDENT sem `has_access` à lesson dona do material.
- [ ] AC4: `MarkCompleteButton.tsx`:
  - Server Action toggleProgress atualiza apenas o user autenticado (anti-IDOR — coberto também em F9.8 mas validar fluxo client → action).
  - UI reflete estado real após mutation (revalidate ou optimistic + reconciliação).
- [ ] AC5: `LockedContent.tsx`:
  - Não renderiza nenhum dado sensível da lesson (description, materials, video_id) — apenas título, cohort name e CTA de matrícula.
- [ ] AC6: `NotificationBell.tsx` + `OnboardingWizard.tsx` — confirmar que não fazem polling sem rate limit nem usam token de auth no client.
- [ ] AC7: `renderContent()` em `src/lib/content` (chamado em `aula/page.tsx:239` para conteúdo MDX/HTML/MARKDOWN) sanitiza HTML antes de renderizar. Tentar injetar XSS via `lesson.content` (admin) → não executa no client.
- [ ] AC8: Bugs P0/P1 CORRIGIDOS na story; P2+ documentados em `docs/smart-memory/agents/research/student-components-audit.md`.

- DOMPurify está em `package.json` (confirmar). Pode haver `@types/dompurify` stub artificial — referência em [memory: project_vercel_deploy].
