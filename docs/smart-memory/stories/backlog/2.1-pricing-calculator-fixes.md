# Story 2.1: PricingCalculator V1 — fixes de squad reset + aria-pressed

## Decisão de lead

Manter V1 (squad tabs, parcelas, trust indicators, richer CRO) e aplicar fixes pontuais. V2 era dead code — menos CRO, deletado intencionalmente na Story 1.1.

## Objetivo

Corrigir 3 problemas no `PricingCalculator` ativo (`src/app/mentoria/pricing-calculator.tsx`):
1. Bug: seleção resetada ao trocar squad (destrói customização do usuário)
2. A11y: squad tabs sem `aria-selected` / `role="tab"`
3. A11y: professional row buttons sem `aria-pressed` / `aria-label`

## Fixes implementados

### Fix 1 — Squad reset bug

**Antes:** `useEffect(() => setSelected(new Set(...)), [activeSquad])` — ao trocar squad, a seleção era destruída e reiniciada com todos marcados.

**Depois:** estado migrado para `selectionBySquad: Record<squadId, Set<profId>>` — cada squad mantém sua própria seleção independente, inicializada na montagem com todos selecionados. Trocar squad preserva a seleção customizada de cada squad.

### Fix 2 — Squad tabs aria

Squad tab buttons agora têm `role="tab"` e `aria-selected={activeSquad === squad.id}`. O container recebeu `role="tablist"` e `aria-label="Tipo de equipe"`.

### Fix 3 — ProfessionalRow aria

Buttons de toggle agora têm `aria-pressed={checked}` e `aria-label` no padrão: `"${label}, R$ ${annual}/ano, selecionado/não selecionado"`.

## Acceptance Criteria

- [x] AC1: Trocar de squad não reseta seleção — cada squad mantém estado próprio.
- [x] AC2: Squad tab buttons têm `role="tab"` e `aria-selected`.
- [x] AC3: Container de tabs tem `role="tablist"` e `aria-label`.
- [x] AC4: ProfessionalRow buttons têm `aria-pressed` e `aria-label` descritivo.
- [x] AC5: `tsc --noEmit` passa sem erros.
- [ ] AC6: Smoke test em `/mentoria#investimento` — selecionar itens no squad Engineering, trocar para Growth, voltar — Engineering mantém a seleção anterior.
- [ ] AC7: Screen reader anuncia estado do tab ativo e estado dos toggles de profissional.
