# Story F5.3: Certificates — geração PDF + página pública de verificação

## Objetivo

Ao concluir 100% das lessons de um curso, gerar certificate PDF via `@react-pdf/renderer` e disponibilizar página pública `/certificado/v/[code]` para verificação.

## Acceptance Criteria

- [ ] AC1: Server Action `checkAndIssueCertificate(courseId, userId)` chamada após `saveProgress` com `completed=true`:
  - Verifica se todas as lessons das cohorts que liberam o course pra esse user estão `completed`
  - Se sim e ainda não há certificate, cria registro em `certificates` com `verification_code` único (formato `AIOX-2026-XXXXXX`)
  - Gera PDF e salva em bucket `certificates` no path `{user_id}/{certificate_id}.pdf`
  - Cria `notifications` "Seu certificado está pronto"
- [ ] AC2: `emails/certificate-ready.tsx` enviado ao aluno
- [ ] AC3: `app/(student)/certificados/page.tsx` lista certificates do usuário com botão de download (signed URL 1h)
- [ ] AC4: `app/(public)/certificado/v/[code]/page.tsx` página pública que mostra:
  - Nome do aluno, nome do curso, data de emissão, código de verificação
  - Selo "Certificado verificado" se `verification_code` existe
  - 404 se code inválido
- [ ] AC5: PDF tem layout próprio: logo, nome do aluno, nome do curso, data, código de verificação, QR code para `/certificado/v/[code]`
- [ ] AC6: Performance: geração de PDF é assíncrona; não bloqueia o `saveProgress`

## Escopo

**IN:**
- Server Action de geração
- Componente PDF
- Página pública de verificação
- Listagem em /certificados
- Email + notification

**OUT:**
- Compartilhamento social (Fase 8 ou follow-up)
- Templates múltiplos de certificate (futuro)

## Contexto Técnico

- `verification_code`: 8-12 chars alfanuméricos, prefixo "AIOX-2026-" (configurável)
- PDF em `@react-pdf/renderer` com fonts customizadas (Fraunces para título)
- QR code via library `qrcode` server-side
- 100% completion check: contar lessons únicas liberadas pro user no course (via `cohort_courses` + `included_module_ids`) vs lessons completadas
