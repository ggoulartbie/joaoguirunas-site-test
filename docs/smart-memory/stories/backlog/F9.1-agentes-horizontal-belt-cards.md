# Story F9.1: /agentes — horizontal card belt + cards menores

## Objetivo
Redesenhar a seção de cards da página `/agentes`: cards com metade do tamanho atual, dispostos em fila horizontal contínua que se move da direita para a esquerda com o scroll — alinhada à animação de câmera dos planetas.

## Acceptance Criteria
- [ ] AC1: Cards têm aprox. metade da largura atual — width fixo ~220px (desktop), ~160px (mobile), com `aspect-square` mantido
- [ ] AC2: Cards de cada squad são dispostos em fila horizontal (flex row), não em grid 2 colunas
- [ ] AC3: Ao scrollar para baixo, os cards se movem horizontalmente da direita para a esquerda (mesma direção da câmera dos planetas)
- [ ] AC4: A animação de entrada/saída é contínua e horizontal — sem saltos abruptos ao mudar de squad
- [ ] AC5: Em mobile, os cards têm scroll horizontal nativo (overflow-x: scroll + snap) como fallback
- [ ] AC6: O card header (squad label, título, descrição) permanece no lado esquerdo como está hoje
- [ ] AC7: Cards mantêm toda info atual — foto, codename, id, title, badges — com tipografia compacta proporcional

## Escopo

**IN:**
- `src/app/agentes/_components/SquadSection.tsx` — substituir `grid-cols-2` por fila horizontal animada
- `src/app/agentes/_components/AgentCard.tsx` — tornar card compacto (menor, layout condensado)
- Criar `src/app/agentes/_components/AgentBelt.tsx` se necessário para isolar a lógica do belt

**OUT:**
- Não alterar `SolarSystemScene.tsx`, `CameraRig.tsx`, `Planet.tsx` — a câmera 3D não muda
- Não alterar o SEO/metadata da página
- Não alterar a SquadSideNav

## Contexto Técnico

### Estado atual (SquadSection.tsx)
```
- Cada SquadSection: min-h-[150vh] + sticky top-[12vh]
- Layout: grid-cols-[1fr_minmax(640px,720px)] — info à esquerda, cards à direita
- Cards: grid-cols-2 com 2.5-4px gap
- Animação: useScroll + framer-motion useTransform
  - cardsX: scrollYProgress [0.05→0.30→0.65→0.90] → ['80%', '0%', '0%', '-60%']
  - Mobile: sem translate X, só fade
```

### Comportamento desejado (novo)
```
- Cards em flex-row, wrapped em container overflow-hidden
- Container fixo à direita (a esquerda fica o header do squad como hoje)
- scrollYProgress → translateX: ['+100%' → '0%' → '-100%'] — entrada da direita, saída à esquerda
- Os cards NÃO ficam em grid — ficam em linha (1 ou 2 linhas, ver layout abaixo)
- Velocidade do translate sincronizada com o PLANET_X da câmera:
  - squad 0 (dev): progressRef 0.20-0.32
  - squad 1 (sites): progressRef 0.42-0.55
  - squad 2 (social): progressRef 0.65-0.78
  - squad 3 (traffic): progressRef 0.88-1.00
```

### Layout dos cards no belt
```
Desktop: 2 linhas × N/2 cards (staggered — linha 2 um pouco mais à direita)
  - Linha 1: cards 0, 2, 4, 6, ...  (índices pares)
  - Linha 2: cards 1, 3, 5, 7, ...  (índices ímpares), com translateX extra +40px
  - gap entre cards: 12px
  - Cada card: w-[220px], aspect-square na imagem

Mobile: scroll horizontal nativo
  - overflow-x: auto, overscroll-x: contain
  - Cards: w-[160px], sem framer-motion animation
  - Sem fila dupla — linha única com snap-x
```

### Transição entre squads — "seguir com a mesma animação do planeta"
```
- A entrada da próxima squad: cardsX parte de '80%' (mesma direção do planeta)
- A saída da squad atual: cardsX vai para '-60%'
- Isso cria continuidade visual: o planeta entra pela direita, os cards também
- NÃO usar fade de opacity entre squads — manter o X slide como único vetor de transição
```

### AgentCard — versão compacta
```
Mudanças no componente:
1. Imagem: aspect-square mas sem height restrito — o w-[220px] do pai define
2. Body (p-3 → p-2): compactar o padding
3. Remover: race label (hidden em mobile já), abilities badges
4. Manter: codename (h3), id (monospace), title
5. Tamanho de fonte: reduzir ~1 step — text-base → text-sm, text-sm → text-xs
6. Hover: manter border accent + scale(1.02)
```
