# Story F8.3: Mobile + acessibilidade

## Objetivo

Audit completo de responsividade mobile e acessibilidade WCAG AA em todas as áreas (público, aluno, admin).

## Acceptance Criteria

- [ ] AC1: Todas as páginas testadas em viewport 360x800 (Android pequeno) e 414x896 (iPhone): sem overflow horizontal, todos os CTAs alcançáveis
- [ ] AC2: VideoPlayer responsivo (16:9 com max-width)
- [ ] AC3: Tabelas no admin com scroll horizontal ou vertical em mobile
- [ ] AC4: Form de cohort no admin com tabs adaptadas para mobile (accordion ou dropdown)
- [ ] AC5: Contraste mínimo 4.5:1 em texto normal, 3:1 em UI components (verificar com axe DevTools)
- [ ] AC6: Navegação por teclado funcional em forms, modals, dropdowns
- [ ] AC7: ARIA labels em ícones (sininho, lock, etc); `aria-current="page"` em nav ativa
- [ ] AC8: Foco visível em todos os elementos interativos (`focus-visible:` nos componentes shadcn)
- [ ] AC9: Skip link para conteúdo principal
- [ ] AC10: Lighthouse a11y score >= 95 em /dashboard, /curso/[slug]/aula/..., /admin

## Escopo

**IN:**
- Audit + correções mobile
- Audit + correções a11y
- Testes Lighthouse

**OUT:**
- A11y AAA (futuro)
- Suporte a leitores de tela exóticos (out of scope)

## Contexto Técnico

- shadcn/ui já vem com bons defaults; foco em ajustes pontuais
- axe-core para automação local
- Lighthouse CI no PR check (configurar em F8.6)
