# Story F9.x: Fix storage — avatars UPDATE sem with check

## Contexto

Identificado pelo QA durante F1.6 como concern não-bloqueante. A policy de UPDATE no bucket `avatars` não tem `WITH CHECK`, o que significa que um usuário poderia potencialmente mover o arquivo para outro path via UPDATE sem que a policy valide o destino.

## Acceptance Criteria

- [ ] AC1: Policy `avatars: update pelo próprio usuário` recriada com `WITH CHECK` que valida que o path de destino também pertence ao `auth.uid()` do usuário
- [ ] AC2: Migration aplicada via `supabase db push`
- [ ] AC3: Documentação em `storage.md` atualizada

## Correção esperada

```sql
drop policy if exists "avatars: update pelo próprio usuário" on storage.objects;

create policy "avatars: update pelo próprio usuário"
  on storage.objects for update
  using (
    bucket_id = 'avatars'
    and auth.uid()::text = (storage.foldername(name))[1]
  )
  with check (
    bucket_id = 'avatars'
    and auth.uid()::text = (storage.foldername(name))[1]
  );
```
