# Story F7.8: /turmas listagem + /turmas/[slug] (LP automática)

## Objetivo

Páginas públicas de catálogo de cohorts: lista as cohorts com `has_public_page=true` e renderiza LP automática para cada uma com descrição, cursos inclusos, preço e CTA de compra.

## Acceptance Criteria

- [ ] AC1: `app/(public)/turmas/page.tsx` lista cohorts com `has_public_page=true` E `is_purchasable=true` ordenadas por preço
- [ ] AC2: `app/(public)/turmas/[slug]/page.tsx` LP automática:
  - Hero com `cover_image_url`, `name`, `description`
  - Seção "O que está incluso": lista cursos via `cohort_courses` com count de módulos/aulas
  - Detalhes: `start_date`, `end_date`, vagas restantes (se `total_seats`), `access_duration_days`, `has_live_sessions`, `has_support`
  - Preço destacado: `entry_price_cents` formatado em BRL + parcelas
  - Cupom: input opcional valida em tempo real
  - CTA "Quero entrar" → dispara checkout (F7.3)
  - FAQ se houver (opcional, virá de campo extra na cohort em follow-up)
  - Selo de garantia (texto institucional)
- [ ] AC3: SEO: metadata dinâmica (title, description, og:image), schema.org Course JSON-LD
- [ ] AC4: Mobile responsive
- [ ] AC5: Estado "Vagas esgotadas" se `filled_seats >= total_seats`: CTA muda para "Lista de espera" (form que captura email — opcional, mínimo: input + Server Action que salva em tabela `waitlist` futura)

## Escopo

**IN:**
- 2 páginas públicas
- LP automática a partir dos campos da cohort
- SEO básico com Course schema
- Validação de cupom client-side (com Server Action)

**OUT:**
- Tabela `waitlist` (criar em follow-up se necessário)
- A/B test de copies (post-MVP)

## Contexto Técnico

- LP é "automática" — admin edita campos da cohort em F6.3 e LP reflete
- Schema Course JSON-LD: usar mesmo padrão da story 3.2 (mentoria) já implementada
- Cupom valida via `validateCoupon(code, cohortId)` Server Action
