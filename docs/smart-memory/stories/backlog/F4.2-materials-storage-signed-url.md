# Story F4.2: Materials — listagem e download via signed URL

## Objetivo

Listar materials de uma lesson e gerar signed URLs de download server-side respeitando `has_access`.

## Acceptance Criteria

- [ ] AC1: Componente `<Materials lessonId>` lista materials da lesson com título, kind (badge: PDF/ZIP/IMG), tamanho
- [ ] AC2: Server Action `getDownloadUrl(materialId)`:
  - Carrega material e seu lesson_id
  - Valida `has_access(user_id, lesson_id)`
  - Para `kind=LINK`: retorna `external_url`
  - Para outros: gera signed URL do bucket `materials` com expiração 5min via `supabase.storage.from('materials').createSignedUrl`
- [ ] AC3: Botão de download dispara o action e abre URL; UX mostra loading durante geração
- [ ] AC4: Materials ordenados por `sort_order`
- [ ] AC5: Estado vazio: "Esta aula não possui materiais"

## Escopo

**IN:**
- Componente listagem
- Server Action de signed URL
- Suporte a tipo LINK (URL externa)

**OUT:**
- Upload pelo admin (Fase 6)
- Watermark em PDFs (futuro)

## Contexto Técnico

- Bucket `materials` é privado (F1.6); signed URL é único caminho de download
- 5min de expiração — adequado para browser fechar antes do link expirar
- ADR-001 D6: materials herdam acesso da lesson
