# Story F3.3: Página da aula + has_access + LockedContent

## Objetivo

Implementar a página `/curso/[slug]/aula/[lesson-slug]` como Server Component que valida acesso via `has_access()` e renderiza player+conteúdo OU `<LockedContent />` com CTA de upsell.

## Acceptance Criteria

- [ ] AC1: `app/(student)/curso/[slug]/aula/[lesson-slug]/page.tsx` é Server Component
- [ ] AC2: Carrega lesson + module + course pelo slug; 404 se não existir
- [ ] AC3: Chama `has_access(user_id, lesson_id)` (RPC ou query SQL direta) no servidor
- [ ] AC4: Se `false`: renderiza `<LockedContent />` com cadeado, descrição da cohort que libera, CTA "Conhecer turma X" → `/turmas/[slug]`. **Não inclui video_id no DOM**
- [ ] AC5: Se `true`: renderiza `<VideoPlayer />` (F3.2) + conteúdo da aula (F3.4) + tabs (Sobre / Materiais / Comentários — F3.5 e F4)
- [ ] AC6: Sidebar/breadcrumb mostra módulos do curso, indicando qual aula está ativa, quais foram completadas (verde) e quais bloqueadas (cadeado pequeno)
- [ ] AC7: Navegação anterior/próxima pulando aulas bloqueadas (ou com indicador "próxima aula é bloqueada")
- [ ] AC8: SEO: `metadata` dinâmica com título da aula e descrição (mas robots noindex porque é conteúdo logado)
- [ ] AC9: Mobile responsive

## Escopo

**IN:**
- Página completa server-side com gating
- LockedContent component
- Sidebar/navegação de módulos
- Metadata + noindex

**OUT:**
- Renderização de MDX/HTML/Markdown (F3.4)
- Comments e materials (F3.5, F4)
- Player real (F3.2 já entregue)

## Contexto Técnico

- **Gating é server-side**: nunca enviar `video_id` para Client Component se `has_access` é false
- `LockedContent` deve ser visualmente atraente (não punitivo) — D15
- Decisão de qual cohort mostrar no CTA: query `cohort_courses` que inclui o módulo/curso da aula, com `is_purchasable=true`, ordenado por preço (mais barata primeiro)
