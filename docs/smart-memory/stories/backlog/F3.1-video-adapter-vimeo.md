# Story F3.1: Video adapter + Vimeo implementation

## Objetivo

Criar abstração de vídeo (`/lib/video/`) com interface neutra e implementação Vimeo, permitindo trocar provider depois sem mexer em UI.

## Acceptance Criteria

- [ ] AC1: `lib/video/interface.ts` define `VideoProvider` com métodos: `getEmbedUrl(videoId, options)`, `uploadVideo(file, metadata)`, `getVideoMetadata(videoId)`
- [ ] AC2: `lib/video/vimeo.ts` implementa Vimeo Standard com domain-locking (whitelist em `VIMEO_DOMAIN_WHITELIST`)
- [ ] AC3: `getEmbedUrl()` retorna iframe URL com domain-locking ativo, hash de privacidade e parâmetros para esconder controles fora do necessário
- [ ] AC4: Vimeo access token só usado server-side (`import 'server-only'`)
- [ ] AC5: Helper `lib/video/index.ts` exporta provider singleton baseado em env (`VIDEO_PROVIDER=vimeo`)
- [ ] AC6: Variáveis de ambiente documentadas no README

## Escopo

**IN:**
- Interface neutra
- Vimeo implementation com domain-locking
- Singleton/factory baseado em env

**OUT:**
- Componente `<VideoPlayer />` (F3.2)
- Watermark dinâmico (Fase 8 ou follow-up)
- Cloudflare Stream implementation (futuro ADR)

## Contexto Técnico

- Domain-locking do Vimeo: whitelist de domínios na conta; embed só funciona se `Referer` bate
- `video_id` da tabela `lessons.video_id` é id do Vimeo
- ADR-001 D5: adapter para troca futura
