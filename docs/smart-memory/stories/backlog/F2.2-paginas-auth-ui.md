# Story F2.2: Páginas de auth — login, cadastro, recuperar senha

## Objetivo

Criar as 4 páginas públicas de autenticação com UI alinhada ao KV do site, validação Zod + React Hook Form, e Server Actions que usam o client Supabase server-side.

## Acceptance Criteria

- [ ] AC1: `app/(public)/login/page.tsx` com form (email, senha) + botão "Entrar com Google" + links pra cadastro e recuperar senha
- [ ] AC2: `app/(public)/cadastro/page.tsx` com form (nome, email, senha, confirmar senha) + Google OAuth
- [ ] AC3: `app/(public)/recuperar-senha/page.tsx` com form (email) que dispara reset via Supabase
- [ ] AC4: `app/(public)/redefinir-senha/[token]/page.tsx` que valida o token e permite criar nova senha
- [ ] AC5: `app/auth/callback/route.ts` (Route Handler) que troca código OAuth por sessão e redireciona pra `/dashboard`
- [ ] AC6: Validação Zod em todos os forms; mensagens de erro em **português** (D18)
- [ ] AC7: Após signup com email/senha, página informa "Confira seu email para confirmar"
- [ ] AC8: Após login bem-sucedido, redireciona pra `/dashboard` (ou pra `redirectTo` se presente em query)
- [ ] AC9: UI usa shadcn/ui (Form, Input, Button) com tokens KV existentes; logo João Guirunas no header
- [ ] AC10: Erros do Supabase traduzidos pra mensagens amigáveis ("Email ou senha inválidos", "Email já cadastrado", etc)
- [ ] AC11: Mobile responsive

## Escopo

**IN:**
- 4 páginas + 1 callback OAuth
- Server Actions: `signIn`, `signUp`, `requestPasswordReset`, `updatePassword`
- Validação e i18n de erros
- UI alinhada ao KV (Fraunces no H1, tokens existentes)

**OUT:**
- Middleware (F2.3)
- Templates de email Resend (F2.4)
- Onboarding wizard (Fase 8)

## Contexto Técnico

- Server Actions chamam `supabase.auth.signInWithPassword`, `signUp`, `resetPasswordForEmail`
- Cookie de sessão setado via `@supabase/ssr` cookie handlers
- `redirectTo` query param permite voltar pra página que disparou login (ex.: usuário tentou comprar, foi mandado pro login, depois volta pro checkout)
- Padrão visual: usar layout existente do site público (header simplificado, sem nav completa)
