### Bug crítico — B-01 (P0 SEGURANÇA)

`changePassword` em `src/app/actions/profile.ts:21-29` chama `supabase.auth.updateUser({ password: newPassword })` **sem revalidar a senha atual**. UI captura `currentPassword` e valida só não-vazio (`PerfilClient.tsx:319-322`), mas nunca envia esse valor para a server action.

**Cenário de exploit:** notebook desbloqueado → atacante abre `/academy/perfil`, preenche qualquer string em "senha atual" e duas senhas iguais nos outros campos → senha alterada. Vítima perde acesso à conta.

**Fix sugerido:**
```typescript
// src/app/actions/profile.ts
export async function changePassword(currentPassword: string, newPassword: string): Promise<{ error?: string }> {
  if (newPassword.length < 8) return { error: 'Nova senha deve ter ao menos 8 caracteres.' }
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()
  if (!user?.email) return { error: 'Sessão inválida.' }
  const { error: signInErr } = await supabase.auth.signInWithPassword({
    email: user.email,
    password: currentPassword,
  })
  if (signInErr) return { error: 'Senha atual incorreta.' }
  const { error } = await supabase.auth.updateUser({ password: newPassword })
  if (error) return { error: 'Erro ao alterar senha.' }
  return {}
}
```
E o caller em `PerfilClient.tsx:334` deve passar `currentPassword`.
