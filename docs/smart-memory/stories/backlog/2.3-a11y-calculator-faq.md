# Story 2.3: Acessibilidade — calculadora (role=checkbox) e FaqAccordion (aria-controls)

## Objetivo
Elevar a semântica de acessibilidade dos dois componentes interativos do `/mentoria` para conformidade WCAG 2.1 AA: (a) migrar os toggles da `PricingCalculator` do padrão `button[aria-pressed]` para `button[role=checkbox][aria-checked]` (mais preciso para seleção múltipla não-mutuamente-exclusiva); (b) garantir que cada botão do `FaqAccordion` tem `aria-controls` apontando ao painel correspondente.

## Dependências

- **BlockedBy:** [[2.1-pricing-calculator-v1-to-v2|Story 2.1]] — esta story modifica o componente canônico `PricingCalculator` resultante da substituição V1→V2. Rodar depois da 2.1 evita conflito de merge e garante que o `aria-pressed` adicionado na 2.1 seja substituído corretamente.

## Acceptance Criteria

- [ ] AC1: No `PricingCalculator` (arquivo canônico após a Story 2.1), cada botão de toggle de profissional recebe `role="checkbox"` e `aria-checked={active}`. O `aria-pressed` adicionado na 2.1 é **removido** (`role=checkbox` + `aria-checked` substitui semanticamente).
- [ ] AC2: O `aria-label` adicionado na 2.1 permanece (ou é ajustado para refletir estado via `aria-checked` sem duplicar a palavra "selecionado" — ver "Nota técnica sobre redundância" abaixo).
- [ ] AC3: Suporte a teclado: botão responde a `Space` para alternar (comportamento nativo de `button` já cobre isso, mas validar que o `role=checkbox` não quebra — ver contexto técnico).
- [ ] AC4: Em `src/app/mentoria/faq-accordion.tsx`, cada botão `<button>` recebe:
  - `id={`faq-trigger-${i}`}` (identificador do trigger).
  - `aria-controls={`faq-panel-${i}`}` (aponta para o painel).
- [ ] AC5: Em `src/app/mentoria/faq-accordion.tsx`, o painel (`<div>` após o botão) recebe:
  - `id={`faq-panel-${i}`}`.
  - `role="region"` e `aria-labelledby={`faq-trigger-${i}`}`.
- [ ] AC6: `pnpm build` conclui sem erros.
- [ ] AC7: Validação com screen reader (VoiceOver ou NVDA, spot-check): calculadora anuncia "caixa de seleção, Marketing Manager, R$ 96.000/ano, marcada" ao receber foco; FAQ anuncia "expandir Preciso saber programar?, botão, expandido/recolhido" e ao expandir lê o painel associado.
- [ ] AC8: Nenhum warning de lint `jsx-a11y` relacionado aos dois componentes.

## Escopo

- Modificar `src/app/mentoria/faq-accordion.tsx`: adicionar `id` + `aria-controls` nos triggers, `id` + `role="region"` + `aria-labelledby` nos painéis.

## Nota técnica sobre redundância

Com `role="checkbox"` + `aria-checked`, alguns screen readers já anunciam o estado naturalmente ("marcada" / "não marcada"). Portanto o `aria-label` pode ser simplificado removendo o sufixo de estado:

```tsx
aria-label={`${professional.label}, R$ ${professional.cost.toLocaleString('pt-BR')}/ano`}
```

O estado fica por conta do `aria-checked`. Isso evita double-reading em leitores que anunciam ambos.

## Notas técnicas

### Fix 1 — `role="checkbox"` no toggle da calculadora

```tsx
<button
  type="button"
  role="checkbox"
  aria-checked={active}
  aria-label={`${professional.label}, R$ ${professional.cost.toLocaleString('pt-BR')}/ano`}
  onClick={onToggle}
  onKeyDown={(e) => {
    if (e.key === ' ' || e.key === 'Enter') {
      e.preventDefault();
      onToggle();
    }
  }}
  className={cn(/* ... */)}
>
```

**Nota sobre keyboard:** Quando um `<button>` recebe `role="checkbox"`, alguns screen readers mudam as expectativas de interação. O handler explícito para `Space`/`Enter` garante cobertura consistente (botões nativos já tratam `Enter` por padrão, mas `Space` pode variar em combinação com role override).

### Fix 2 — FaqAccordion `aria-controls`

Estado atual (`src/app/mentoria/faq-accordion.tsx`):
- `aria-expanded` presente (bom).
- Painel renderizado condicionalmente com `{openIndex === i && (...)}` — não tem `id`.

Código-alvo:

```tsx
<button
  onClick={() => setOpenIndex(openIndex === i ? null : i)}
  className="..."
  aria-expanded={openIndex === i}
  aria-controls={`faq-panel-${i}`}
  id={`faq-trigger-${i}`}
>
  {/* ... */}
</button>
{openIndex === i && (
  <div
    id={`faq-panel-${i}`}
    role="region"
    aria-labelledby={`faq-trigger-${i}`}
    className="..."
  >
    {item.a}
  </div>
)}
```

**Alternativa considerada:** renderizar o painel sempre (sem conditional) com `hidden={openIndex !== i}` — isso melhora a descoberta por screen readers que navegam pela DOM antes da expansão. Fora de escopo para esta story (mudança de comportamento visual), mas é candidato a story futura.

- **Risco:** baixo. Mudanças são aditivas/semânticas; não afetam lógica ou visual.
