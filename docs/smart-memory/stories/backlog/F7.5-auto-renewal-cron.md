# Story F7.5: Renovação automática (opt-in) + cron diário

## Objetivo

Implementar opt-in de renovação automática via Stripe Subscription Schedule e cron Vercel diário que: marca expirados, envia lembretes e processa renovações pendentes.

## Acceptance Criteria

- [ ] AC1: `/perfil` tem toggle "Renovação automática" por matrícula:
  - ON: cria Stripe Subscription Schedule programada para `expires_at - 7 dias`; salva `cohort_members.next_renewal_at`; `auto_renew_enabled=true`
  - OFF: cancela Subscription Schedule; `auto_renew_enabled=false`
- [ ] AC2: Toggle só aparece se `cohorts.allows_auto_renewal=true`
- [ ] AC3: `app/api/cron/daily/route.ts` (Vercel Cron, diário às 09:00 BRT):
  - **(a) Expirar memberships**: UPDATE `cohort_members SET status='EXPIRED' WHERE status='ACTIVE' AND expires_at < now()`
  - **(b) Lembretes**: para `cohort_members ACTIVE` com `expires_at` em 15, 7 e 3 dias: enviar email "Sua matrícula expira em N dias — renove agora" (template em F7.6)
  - **(c) Processar renovações pendentes**: Stripe Subscription Schedule já dispara automaticamente; cron só verifica `cohort_members PAST_DUE` por mais de 3 dias e desativa
- [ ] AC4: Endpoint protegido por `Authorization: Bearer ${CRON_SECRET}` (Vercel Cron header)
- [ ] AC5: `vercel.json` configurado com `{"crons": [{"path": "/api/cron/daily", "schedule": "0 12 * * *"}]}` (12 UTC = 09 BRT)
- [ ] AC6: Idempotência: re-rodar mesmo dia não duplica emails (flag `last_reminder_sent_at` em `cohort_members`)
- [ ] AC7: Logs Sentry estruturados

## Escopo

**IN:**
- Toggle UI no /perfil
- Stripe Subscription Schedule
- Cron diário com 3 tarefas
- Auth do cron

**OUT:**
- Templates de email (F7.6)
- Página /perfil completa (F7.7)

## Contexto Técnico

- ADR-001 D13: opt-in via Subscription Schedule
- Subscription Schedule da Stripe: cria phase futura que cobra automaticamente
- Vercel Cron: configurar em `vercel.json`; só roda em produção (testar localmente via curl com auth header)
- Coluna nova: `cohort_members.last_reminder_sent_at timestamptz` — adicionar via migration follow-up
