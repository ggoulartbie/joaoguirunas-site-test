# Story F5.1: Fórum — categorias, threads e replies

## Objetivo

Implementar `/forum` como fórum geral único com categorias, listagem de threads, criação, replies em árvore (1 nível), votos e marcação de resposta aceita.

## Acceptance Criteria

- [ ] AC1: `app/(student)/forum/page.tsx` lista categorias ativas + threads recentes globais
- [ ] AC2: `app/(student)/forum/categoria/[slug]/page.tsx` lista threads da categoria, ordenados por `last_activity_at desc`, com paginação
- [ ] AC3: `app/(student)/forum/thread/[slug]/page.tsx` mostra thread + replies em árvore (1 nível)
- [ ] AC4: Server Actions: `createThread(categoryId, title, content)`, `createReply(threadId, content, parentReplyId?)`, `vote(threadId|replyId)`, `markAsAccepted(replyId)` (apenas autor da thread), `markAsResolved(threadId)`
- [ ] AC5: Editor de texto com Markdown (Markdown nativo nos textareas, preview opcional)
- [ ] AC6: Upvote em thread e em reply (tabela `votes`); count agregado mostrado
- [ ] AC7: Thread com `is_resolved=true` mostra badge "Resolvida" no card
- [ ] AC8: Reply marcada como `is_accepted_answer=true` aparece destacada no topo das replies
- [ ] AC9: Busca simples (input em `/forum`) filtra threads por título/content via `ilike`
- [ ] AC10: Acesso ao fórum exige usuário com **alguma** cohort_member ACTIVE (verificação em layout)
- [ ] AC11: Admin/Mentor podem deletar (soft) threads e replies; pin de thread

## Escopo

**IN:**
- 3 páginas + Server Actions
- Sistema de votos
- Marcação de resposta aceita
- Pin/resolved
- Busca simples
- Editor Markdown

**OUT:**
- Categorias por cohort (D7: fórum geral único)
- Notificação de reply (Fase 8)
- Editor WYSIWYG TipTap (admin usa, alunos usam textarea)
- Pesquisa full-text avançada (post-MVP)

## Contexto Técnico

- ADR-001 D7: fórum único, categorias tópicas (não por cohort)
- Slug de thread: gerado de `title` + sufixo (id curto)
- `last_activity_at` atualizado em cada novo reply (trigger ou via Server Action)
- View count: incrementar em GET da página, debounce client-side
