# Story F2.3: Middleware Next.js + helpers RBAC

## Objetivo

Implementar middleware que valida sessão Supabase em rotas protegidas e helpers `requireUser`, `requireRole`, `requireAdmin` para uso em Server Components e Server Actions.

## Acceptance Criteria

- [ ] AC1: `middleware.ts` na raiz do projeto que:
  - Atualiza sessão Supabase a cada request via `@supabase/ssr`
  - Em rotas `/dashboard/*`, `/meus-cursos/*`, `/curso/*`, `/forum/*`, `/agenda/*`, `/certificados/*`, `/perfil/*`: redireciona pra `/login?redirectTo=...` se não autenticado
  - Em rotas `/admin/*`: redireciona pra `/login` se não autenticado E retorna 403 se autenticado mas role != ADMIN
- [ ] AC2: `lib/auth/server.ts` exporta:
  - `getCurrentUser()` → `Profile | null`
  - `requireUser()` → `Profile` ou `redirect('/login')`
  - `requireRole(roles: Role[])` → `Profile` ou throw 403
  - `requireAdmin()` → `Profile` (ADMIN) ou throw 403
- [ ] AC3: Helpers buscam profile com role do Postgres (não confiar só em metadata do Supabase auth)
- [ ] AC4: Acessar `/admin` sem login redireciona pra `/login`
- [ ] AC5: Acessar `/admin` como STUDENT retorna 403 (page.tsx custom de erro 403)
- [ ] AC6: Acessar `/dashboard` sem login redireciona pra `/login?redirectTo=/dashboard`
- [ ] AC7: Após login bem-sucedido com `redirectTo` na query, vai pra rota correta
- [ ] AC8: Performance: middleware adiciona < 50ms ao request (medir com Vercel Analytics local)

## Escopo

**IN:**
- `middleware.ts` com matcher correto (excluir `/_next`, `/api/webhooks`, public assets)
- Helpers em `lib/auth/server.ts`
- Página de erro 403 customizada
- Testes manuais dos cenários de redirect

**OUT:**
- Helpers de banco para `has_access` (Fase 3)
- UI de "logado como" no header (Fase 8 ou parte da F2.2)

## Contexto Técnico

- `middleware.ts` roda em Edge runtime — usar `@supabase/ssr` (compatível)
- `getCurrentUser()` faz JOIN com `profiles` para pegar role; cachear via React `cache()` por request
- `redirect()` de `next/navigation` em Server Components; em Server Actions usar `redirect()` também
- Webhook Stripe (`/api/webhooks/stripe`) **não pode** passar pelo middleware de auth — adicionar exceção no matcher
- Padrão: helpers lançam erros tipados; layout intercepta e renderiza 403/401
