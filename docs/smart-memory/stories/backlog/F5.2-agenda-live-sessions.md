# Story F5.2: /agenda + live_sessions

## Objetivo

Página `/agenda` com encontros ao vivo das cohorts ativas do aluno, libera `meeting_url` server-side apenas 30min antes do horário, exporta `.ics`.

## Acceptance Criteria

- [ ] AC1: `app/(student)/agenda/page.tsx` lista live_sessions futuras das cohorts onde o aluno está ACTIVE
- [ ] AC2: Sessions passadas com `recording_url` aparecem em seção separada "Encontros gravados"
- [ ] AC3: Card de session mostra: título, data/hora, duração, descrição, cohort de origem
- [ ] AC4: Botão "Entrar no encontro" só renderiza `meeting_url` quando `now() >= scheduled_at - 30min` (verificação **server-side**)
- [ ] AC5: Antes de 30min: botão desabilitado com tooltip "Disponível 30 minutos antes"
- [ ] AC6: Botão "Adicionar à agenda (.ics)" gera arquivo iCalendar com title, description, dtstart, dtend
- [ ] AC7: Recording: link aparece em sessions passadas se `recording_url` preenchido
- [ ] AC8: Mobile responsive
- [ ] AC9: Lembrete: dashboard mostra próximo encontro (já especificado em F3.5)

## Escopo

**IN:**
- Página agenda
- Lógica de exposição condicional do meeting_url
- Geração de .ics
- Listagem de gravações

**OUT:**
- Notificação push 30min antes (Fase 8)
- Email de lembrete de session (Fase 8)
- Upload da gravação automático (admin manual em Fase 6)

## Contexto Técnico

- Server Component compara `now()` no servidor; nunca confiar em client clock
- `.ics` gerado server-side via Route Handler ou library `ics`
- `meeting_url` pode ser Meet, Zoom, etc — plataforma só hospeda o link
