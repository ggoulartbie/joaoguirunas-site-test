# Story F4.1: Sistema de comments nas aulas

## Objetivo

Implementar comments nas aulas com threads de 1 nível, soft delete, edição limitada e badge especial para Mentor/Admin.

## Acceptance Criteria

- [ ] AC1: Componente `<Comments lessonId>` em Server Component que carrega comments via Supabase respeitando RLS
- [ ] AC2: Server Action `createComment(lessonId, content, parentCommentId?)`:
  - Valida `has_access(user_id, lesson_id)`
  - Insere com sanitização de content (DOMPurify)
- [ ] AC3: Reply em comment existente cria nova linha com `parent_comment_id`; UI limita a 1 nível
- [ ] AC4: Soft delete (`deleted_at`) preserva thread; UI mostra "Comentário removido"
- [ ] AC5: Edição até 15min após criação (autor); ADMIN edita qualquer
- [ ] AC6: Badge visual "Mentor" / "Admin" / "Suporte" ao lado do nome quando `profile.role` for esse
- [ ] AC7: Pin de comment por ADMIN/MENTOR (`is_pinned`); pinned aparecem no topo
- [ ] AC8: Markdown básico no content (negrito, link, código inline)
- [ ] AC9: Avatar do autor + nome + timestamp relativo ("há 3 horas")
- [ ] AC10: Mobile responsive

## Escopo

**IN:**
- Componente + Server Actions
- Threads 1 nível
- Soft delete, edição, pin
- Badge de role
- Markdown básico

**OUT:**
- Notificação "alguém respondeu seu comment" (Fase 8)
- Reações/likes (fora do MVP)

## Contexto Técnico

- Comments só visíveis se aluno tem `has_access` da lesson — RLS garante (F1.5)
- Sanitização: DOMPurify server-side antes do INSERT
- Janela de 15min para edição = reduz disputa; após isso, autor pode deletar (soft) e recriar
