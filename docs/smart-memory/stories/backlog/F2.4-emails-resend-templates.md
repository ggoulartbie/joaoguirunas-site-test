# Story F2.4: Emails transacionais — Resend + React Email

## Objetivo

Configurar Resend como provedor de emails e criar templates iniciais (boas-vindas, recuperação de senha) usando React Email.

## Acceptance Criteria

- [ ] AC1: Domínio verificado no Resend (`mail.joaoguirunas.com` ou similar) com SPF/DKIM
- [ ] AC2: `RESEND_API_KEY` e `RESEND_FROM_EMAIL` em `.env.local` e `.env.example`
- [ ] AC3: `lib/email/send.ts` com função `sendEmail({ to, subject, react })` que usa Resend SDK
- [ ] AC4: `emails/welcome.tsx` (React Email) — email de boas-vindas pós-cadastro
- [ ] AC5: `emails/password-reset.tsx` — email de recuperação de senha (se Supabase nativo for substituído por custom; senão usar template do Supabase neste momento)
- [ ] AC6: Template usa logo, paleta e tipografia (Fraunces) do KV do site
- [ ] AC7: Fallback texto puro (Resend gera automático mas validar)
- [ ] AC8: Server Action `signUp` (de F2.2) dispara `welcome.tsx` após confirmação de email (via webhook do Supabase OU trigger no banco)
- [ ] AC9: Documentação em `docs/smart-memory/agents/marketing/email-templates.md` listando templates e quando são disparados
- [ ] AC10: Emails são enviados em ambiente local via Resend "test mode" sem queimar quota; staging/prod usam keys reais

## Escopo

**IN:**
- Setup Resend
- 2 templates iniciais
- Helper `sendEmail`
- Disparo do welcome após signup confirmado

**OUT:**
- Templates de pagamento, lembrete de session, certificate (vão nas fases respectivas)
- Email de expiração próxima (Fase 7 — junto com cron)

## Contexto Técnico

- React Email roda em Node — não importar em Client Component
- Tabela `notifications` é separada (in-app); email é canal complementar
- Para disparar email após confirmação do Supabase, opções:
  1. Webhook de auth (Supabase Auth Hooks)
  2. Trigger no Postgres em `auth.users` (mas evitar lógica pesada em trigger)
  3. Polling em Server Action ao primeiro login pós-confirmação
- Decisão recomendada: trigger `on_auth_user_created` (já existe) **só cria profile**; email de boas-vindas dispara em Server Action no primeiro login
