# Story 1.1: Remover dead code — HeroSection e PricingCalculatorV2

## Objetivo

- [x] AC2: Pasta `src/shared/components/sections/` removida se ficar vazia após o AC1.
- [x] AC3: Arquivo `src/app/mentoria/pricing-calculator-v2.tsx` removido do repositório.

## Contexto Técnico

**Por que é dead code:**
- `HeroSection.tsx` (`src/shared/components/sections/HeroSection.tsx:17`) exporta `HeroSection` mas não é importado em nenhum lugar do código-fonte. Usa accent `#E8601C` divergente do padrão `#FF4400`. A home usa `animated-hero.tsx` (`src/shared/components/ui/animated-hero.tsx`).
- `pricing-calculator-v2.tsx` (`src/app/mentoria/pricing-calculator-v2.tsx:137`) exporta `PricingCalculatorV2` mas `src/app/mentoria/page.tsx:8` importa apenas `PricingCalculator` de `./pricing-calculator` (v1). O V2 nunca é renderizado.

**Módulos afetados:** apenas `src/shared/components/sections/` (removido) e `src/app/mentoria/` (um arquivo a menos). Nenhum runtime impactado.

**Constraints:**
- Não existe test runner no projeto (ver [[../../project/overview]] §Pontos de atenção item 3). Validação é via `pnpm build` + smoke visual em dev (`/` e `/mentoria`).
- Referências em `docs/smart-memory/` devem ser atualizadas para manter o índice semântico consistente.
