# Story 4.3: KV Mentoria — facilitadores rect style + section badges unificados

## Objetivo
Corrigir os dois pontos da /mentoria que mais divergem do KV: as fotos circulares com ring laranja (contrário à direção sharp/cinematográfica) e os section badges com fundo laranja preenchido (viola a regra "laranja máx 8%").

## Acceptance Criteria
- [x] AC1: **Facilitadores — frames retangulares**: removido `rounded-full`, `ring-[3px]`, `ring-[#FF4400]/20`. Substituído por frame com `border: 1px solid rgba(255,255,255,0.12)` (hairline) e hover `box-shadow: 0 0 32px rgba(255,58,14,0.18)` + `border-color: rgba(255,58,14,0.35)` — transição `duration-500`
- [x] AC2: **Facilitadores — foto style**: `rounded-full` removido da `<Image>`. Mantido `grayscale group-hover:grayscale-0 transition-all duration-500`
- [x] AC3: **Facilitadores — tamanho**: mantido `w-44 h-44 sm:w-56 sm:h-56 lg:w-72 lg:h-72`, quadrado
- [x] AC4: **Section badges unificados**: `SectionBadge` refatorado — sem container visual, apenas `<p>` mono com `font-size: 11px`, `letter-spacing: 0.16em`, `color: rgba(255, 58, 14, 0.85)`, `text-transform: uppercase`
- [x] AC5: Todos os `<SectionBadge>` da página mentoria (facilitadores, FAQ) usando o novo componente
- [x] AC6: Visual regression check: layout não quebra em mobile e desktop

## Escopo

**IN:**
- `src/app/mentoria/page.tsx` — facilitadores section + SectionBadge component
- Apenas estilo/layout — nenhum copy é alterado

**OUT:**
- Conteúdo das bios, nomes, links LinkedIn
- RevosForm, PricingCalculator, FaqAccordion (componentes externos)
- Imagens das fotos (src, alt)

## Contexto Técnico

**Facilitadores atual** (linhas 241-259 de mentoria/page.tsx):
```tsx
<div className="w-44 h-44 ... rounded-full overflow-hidden ring-[3px] ring-[#FF4400]/20 group-hover:ring-[#FF4400]/50 ...">
  <Image ... className="... grayscale group-hover:grayscale-0 ..." />
</div>
```

**Facilitadores target**:
```tsx
<div className="w-44 h-44 ... overflow-hidden transition-all duration-500"
  style={{ border: '1px solid rgba(255,255,255,0.16)' }}
  // hover via group: border-color rgba(255,58,14,0.35) + box-shadow
>
  <Image ... className="w-full h-full object-cover object-top grayscale group-hover:grayscale-0 transition-all duration-500" />
</div>
```

**SectionBadge atual**:
```tsx
<div className="inline-flex items-center gap-2 border border-[#FF4400]/30 bg-[#FF4400]/10 px-3 sm:px-4 py-1.5 mb-4 sm:mb-6">
  <span className="text-[#FF4400] text-xs sm:text-[0.65rem]" style={{ fontFamily: "...", textTransform: 'uppercase', letterSpacing: '0.12em', fontWeight: 600 }}>
    {label}
  </span>
</div>
```

**SectionBadge target**: remover o div externo com border/bg. Apenas um `<p>` ou `<span>` mono sem container visual. O ember fica na tipografia, não no bloco.

**KV Rule 02**: *"Nunca duas cores saturadas adjacentes."* O badge com bg laranja + texto laranja + border laranja é exatamente o que violar essa regra.
