# Story F9.18: P1-cleanup F9.12 — bugs B-02 a B-06

## Objetivo
Resolver os 5 bugs P1 abertos identificados pelo sites-qa durante o bug-hunt F9.12 da área do aluno: link Stripe `/test/` em produção, link `/turmas/` inexistente, cross-module navigation sem gate, e duas N+1 queries no fórum.

## Acceptance Criteria

- [ ] **AC1 (B-02 — Stripe test link):** `src/app/(academy)/academy/(student)/perfil/PerfilClient.tsx:658` deve usar `https://dashboard.stripe.com/payments` em produção (não `/test/payments`). Solução: detectar ambiente via `process.env.NODE_ENV === 'production'` ou `NEXT_PUBLIC_STRIPE_MODE` e alternar a URL. Testar em dev (continua `/test/`) e em prod (vai para dashboard live).

- [ ] **AC2 (B-03 — LockedContent → /turmas link):** `src/components/student/LockedContent.tsx:26` aponta para `/turmas/${cohortSlug}` que não existe como rota pública. Substituir por destino válido — opções: (a) `/curso-online` (rota pública atual de venda), (b) `/academy/checkout/${cohortSlug}` se houver cohort comprável, ou (c) remover o link e mostrar texto estático "Esta turma não está mais disponível para compra". Decisão: avaliar com o lead qual opção alinha com o produto, e implementar.

- [ ] **AC3 (B-04 — Cross-module navigation sem gate):** `src/app/(academy)/academy/(student)/curso/[slug]/aula/[lesson-slug]/page.tsx:91-135` calcula `globalPrev`/`globalNext` a partir de `allModulesRaw` (todos os módulos do curso) sem filtrar por `accessibleModuleIds` da matrícula. Resultado: o botão "Próxima aula" pode levar para uma aula em módulo bloqueado, onde `has_access` retorna false e o aluno cai em `LockedContent` (bloqueia AC4 da F9.12). Fix: aplicar o pattern canônico de `src/app/(academy)/academy/(student)/meus-cursos/page.tsx:65-71` para resolver `accessibleModuleIds` do `cohort_courses` e filtrar `allLessons` retendo apenas lessons em módulos acessíveis ANTES de calcular `globalIndex`/`globalPrev`/`globalNext`. Edge case: aluno na última aula liberada deve ver `globalNext = null` (botão oculto/desabilitado), não um link para módulo bloqueado.

- [ ] **AC4 (B-05 — N+1 forum index):** `src/app/(academy)/academy/(student)/forum/page.tsx:48-57` faz uma query de `count` por categoria (N queries) e linhas 83–116 fazem 2 queries de count por thread (2*N queries). Fix: substituir por uma única query que retorne os counts agregados (ex: RPC SQL ou view). Sugestão: criar view `forum_categories_with_counts` e `forum_threads_with_counts` no Supabase, ou usar `select` com `forum_replies(count)` e `votes(count)` no mesmo query usando aggregate references do PostgREST. Critério: Network tab da página `/academy/forum` deve mostrar no máximo 3 queries Supabase (categories, threads, membership) — não N+1.

- [ ] **AC5 (B-06 — N+1 forum detail):** `src/app/(academy)/academy/(student)/forum/[category]/[slug]/page.tsx:114–120+` faz query de count de votos por reply (N queries). Fix: query agregada análoga ao AC4. Critério: página `/academy/forum/[category]/[slug]` com 20 replies deve fazer no máximo 4 queries Supabase (thread, replies, votes-thread, votes-replies-aggregate) — não 1+N.

- [ ] **AC6 (regressão):** Smoke manual após fix:
  - `/academy/perfil` (com pagamento) → link "Ver no Stripe" abre dashboard correto por ambiente
  - `/academy/curso/[slug]/aula/[lesson-slug]` em módulo bloqueado → link "Voltar para turmas" leva a destino válido (não 404)
  - Navegar pela última aula do último módulo liberado → botão "próxima" oculto ou desabilitado (não pula para módulo bloqueado)
  - `/academy/forum` carrega em <500ms server time com 20 threads
  - `/academy/forum/[cat]/[slug]` com 20+ replies carrega em <500ms server time

## Escopo

**IN:**
- Fix nos 5 arquivos identificados (PerfilClient.tsx, LockedContent.tsx, lesson page, forum page, forum detail page)
- Migration ou view SQL se necessária para resolver AC4/AC5 com aggregate query
- Smoke manual cobrindo AC6
- Atualização de F9.12 com os bugs marcados como resolvidos

**OUT:**
- Refactor amplo do fórum (paginação, infinite scroll, cache) — manter foco nos 5 bugs
- Reescrita do componente LockedContent — apenas o link
- Mudança de provider de pagamento — apenas a URL do dashboard
- Otimização de outras páginas com potencial N+1 (se houver, abrir nova story)
- Fixes nos bugs P0 (B-01) ou P2 — fora deste cleanup
