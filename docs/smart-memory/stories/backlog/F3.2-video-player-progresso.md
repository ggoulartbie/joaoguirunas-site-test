# Story F3.2: VideoPlayer + Server Action saveProgress

## Objetivo

Componente `<VideoPlayer />` que embute Vimeo e salva progresso do aluno a cada 10s; conclusão automática ao chegar no fim.

## Acceptance Criteria

- [ ] AC1: `components/student/VideoPlayer.tsx` (Client Component) recebe `lessonId`, `videoId`, `videoProvider`
- [ ] AC2: Embed via `getEmbedUrl()` do adapter F3.1
- [ ] AC3: Player escuta eventos `timeupdate` do Vimeo Player API e dispara `saveProgress(lessonId, seconds)` a cada 10s
- [ ] AC4: Server Action `saveProgress(lessonId, seconds)` em `app/actions/progress.ts`:
  - Verifica `has_access(user_id, lesson_id)` server-side
  - Faz `upsert` em `lesson_progress` (unique user_id + lesson_id)
  - Se `seconds >= duration_seconds * 0.95`, marca `completed=true` e `completed_at=now()`
- [ ] AC5: Botão manual "Marcar como concluída" também dispara o action com `completed=true`
- [ ] AC6: Performance: debounce de 10s reduz tráfego; throttle se evento dispara mais frequente
- [ ] AC7: Indicador visual de % assistido na UI

## Escopo

**IN:**
- Componente player
- Server Action de progresso
- Conclusão automática + manual
- UI de progresso

**OUT:**
- Geração de certificado ao completar curso (Fase 5)
- Próximo/anterior navigation (parte da F3.3)

## Contexto Técnico

- Vimeo Player API JS para timeupdate event
- `lesson_progress.seconds_watched` = max(atual, salvo) — não diminuir
- 95% do duration = "assistiu" (margem para créditos)
