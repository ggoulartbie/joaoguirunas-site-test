# Story 1.1: Fix Vimeo aspect ratio — iframe cortado fora do container 16:9

## Objetivo
Garantir que o iframe do Vimeo (e YouTube) ocupe 100% do container 16:9 sem
overflow visual e sem cortar a barra de controles, independentemente do aspect
ratio do vídeo de origem.

## Acceptance Criteria

- [x] **AC1:** O iframe do Vimeo cabe 100% dentro do container 16:9 (`width: 100%; height: 100%`)
  independente do aspect ratio nativo do vídeo (testar com 16:9, 4:3 e 9:16).
- [x] **AC2:** A barra de controles do Vimeo é visível dentro do container — nenhum
  pixel do player é cortado pelas bordas.
- [x] **AC3:** Sem overflow visual: container com `overflow: hidden` permanece, mas
  o iframe deixa de ser maior que o container.
- [x] **AC4:** Mesmo fix aplicado ao container do YouTube (consistência de markup —
  ambos viram wrappers `position: relative` com iframe `position: absolute; inset: 0`).
- [x] **AC5:** A funcionalidade existente do player permanece intacta: autoplay
  (no YouTube após clique no thumbnail), saveProgress a cada 10s, markComplete
  em 95%, controles do Vimeo (responsive `true`, sem títulos/byline/portrait).
- [x] **AC6:** Sem regressão em layouts mobile (testar 360px, 768px, 1024px, 1440px).

## Escopo

**IN:**
- `src/components/student/VideoPlayer.tsx` — wrapper Vimeo (linhas 298-306)
- `src/components/student/VideoPlayer.tsx` — wrapper YouTube ativo (linhas 273-281)
- Aplicar padrão `padding-top: 56.25%` (ou `aspect-ratio: 16/9` no wrapper) + iframe

- Vimeo Player SDK continua sendo instanciado via `new Player(container, {...})` —

**OUT:**

- Mudanças no thumbnail/play button do YouTube (linhas 241-269) — funciona OK.
- Refactor da estrutura do componente para hooks separados (Vimeo/YouTube).
- Suporte a outros aspect ratios além de 16:9 (vertical, square) — fora do escopo.

## Contexto Técnico

**Bug:** O container atual tem `style={{ aspectRatio: '16/9' }}` mas o iframe que
o Vimeo Player SDK injeta dentro dele assume o aspect ratio nativo do vídeo,
não o do container. Resultado: vídeos 4:3 ou vertical extrapolam o container
e cortam a barra de controles.

**Solução clássica de embed responsivo:**

```tsx
<div className={className}>
  <div
    className="relative w-full overflow-hidden"
    style={{ aspectRatio: '16/9', background: 'var(--void)' }}
  >
    <div
      ref={vimeoContainerRef}
      className="absolute inset-0 h-full w-full"
    />
  </div>
</div>
```

O Vimeo Player SDK detecta o tamanho do container pai (com `responsive: true`)
e injeta um iframe `width: 100%; height: 100%` dentro dele. Forçar o wrapper
interno a `position: absolute; inset: 0` garante que ele preencha o slot 16:9
independente do que o SDK decidir.

Para YouTube (linhas 273-281), aplicar o mesmo padrão — `window.YT.Player(el, ...)`
substitui o `<div>` por um `<iframe>`, então o wrapper deve estar com o sizing
correto antes do mount.
