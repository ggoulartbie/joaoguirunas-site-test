# Story F4.1: Sistema de comments nas aulas

## Objetivo

Implementar comments nas aulas com threads de 1 nível, soft delete, edição limitada e badge especial para Mentor/Admin.

## Acceptance Criteria

- [x] AC1: `CommentsSection.tsx` carrega comments via Supabase
- [x] AC2: Server Actions `addComment`, `editComment`, `deleteComment` com validação de acesso
- [x] AC3: Replies com `parent_comment_id`; UI limita a 1 nível
- [x] AC4: Soft delete (`deleted_at`) — UI mostra "Comentário removido"
- [x] AC5: Edição limitada no tempo (implementada nos actions)
- [x] AC6: Badge visual por role (Mentor/Admin) ao lado do nome
- [x] AC7: `is_pinned` — pinned comments destacados no topo
- [ ] AC8: Markdown básico no content — não verificado explicitamente
- [x] AC9: Avatar inicial + nome + timestamp relativo
- [x] AC10: Mobile responsive
