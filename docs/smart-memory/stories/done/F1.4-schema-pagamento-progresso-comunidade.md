# Story F1.4: Migration — Pagamento + Progresso + Comunidade + Auxiliares

## Objetivo

Completar o schema com as tabelas restantes: pagamentos, cupons, webhook_events, progresso, certificados, comments, fórum (categories/threads/replies/votes) e notifications.

## Acceptance Criteria

- [x] AC1: `payments` com `purchase_kind`, `status`, Stripe IDs, FK para `cohort_members(id)`, `stripe_checkout_session_id` UNIQUE
- [x] AC2: `coupons` por cohort, com `discount_kind`, `applies_to`, unique (cohort_id, code), validade temporal
- [x] AC3: `webhook_events` com `stripe_event_id` UNIQUE (idempotência), payload JSONB, success boolean, error_message
- [x] AC4: `lesson_progress` com unique (user_id, lesson_id), `seconds_watched`, `completed`, `completed_at`
- [x] AC5: `certificates` com `verification_code` UNIQUE, FKs para user/course/cohort/membership, `pdf_storage_path`
- [x] AC6: `comments` com self-reference `parent_comment_id` (threads de 1 nível), soft delete
- [x] AC7: `forum_categories`, `forum_threads` (slug UNIQUE), `forum_replies` (self-reference), `votes` (CHECK target exclusivo; UNIQUEs separados)
- [x] AC8: `notifications` com `notification_type`, `read_at`
- [x] AC9: Índices em queries quentes: `payments(user_id, status)`, `lesson_progress(user_id, completed)`, `forum_threads(category_id, last_activity_at DESC)`, `notifications(user_id, read_at)`
- [x] AC10: Tipos TS regenerados; `supabase db diff` vazio
