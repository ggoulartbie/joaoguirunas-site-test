# Story F8.4: Sentry + Vercel Analytics

## Objetivo

Instrumentar a plataforma com Sentry (erros + traces) e Vercel Analytics (Web Vitals + Speed Insights).

## Acceptance Criteria

- [x] AC1: `@sentry/nextjs` instalado e configurado para client + server + edge
- [x] AC2: `sentry.client.config.ts` com tracing configurado (sampleRate 0.1)
- [x] AC3: `sentry.server.config.ts` configurado; webhook handlers e Server Actions críticas marcadas com tags
- [x] AC4: `Sentry.captureException` em todos catch blocks de Server Actions e Route Handlers
- [x] AC5: Source maps enviados no build (Vercel integration — `withSentryConfig` em next.config.ts)
- [x] AC6: Vercel Analytics + Speed Insights instalados (`@vercel/analytics`, `@vercel/speed-insights`)
- [ ] AC7: Alertas Sentry: configurados no painel sentry.io (requer DSN real — fora do código)
- [x] AC8: Performance: `beforeSend` filtra payload webhook em server config; cookies e headers PII removidos

## Escopo

**IN:**
- Setup Sentry
- Tags e contexts em pontos críticos
- Alertas configurados

**OUT:**
- Logging estruturado custom (Sentry breadcrumbs cobrem)
- Datadog/NewRelic (Sentry+Vercel é suficiente)

## Contexto Técnico

- ADR-001 D2.17: Sentry + Vercel Analytics
- Cuidado com PII: filtrar emails, tokens e dados sensíveis em `beforeSend`
