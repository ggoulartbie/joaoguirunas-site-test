# Story F9.5: Auditar APIs em /api/admin/*

## Objetivo
Garantir que todas as rotas em `src/app/api/admin/*` (a) exigem `requireAdmin()`, (b) validam payload com Zod (ou equivalente), (c) retornam status codes corretos (200/201/400/401/403/404/409/500), e (d) tratam erros sem vazar stack trace.

## Acceptance Criteria
- [x] AC1: Para cada rota em `src/app/api/admin/**/route.ts`, confirmar que a primeira linha do handler executa `await requireAdmin()` (ou helper equivalente). Listar arquivos auditados em File List.
- [x] AC2: Rota `/api/admin/turmas/[id]/resync-stripe` (única atualmente existente) é testada manualmente com (a) sem auth → 401, (b) STUDENT autenticado → 403/redirect, (c) ADMIN com `id` válido → 200 e cohort sincronizada, (d) ADMIN com `id` inválido → 404.
- [x] AC3: Payloads de POST/PATCH são validados com `zod` antes de tocar Supabase; erro de validação retorna 400 com `{ error: 'Dados inválidos', issues: [...] }` (não 500). — N/A: rota não tem body.
- [x] AC4: Erros internos são logados (server-side) mas a response ao cliente não vaza `stack` nem detalhes do schema do banco.
- [x] AC5: Documentação curta em `docs/smart-memory/agents/research/api-admin-inventory.md` lista cada rota, método, payload, response e quem chama (página/componente).

## Escopo

**IN:**
- Auditoria de todas as rotas em `src/app/api/admin/`
- Adicionar `requireAdmin()` onde estiver faltando
- Adicionar validação Zod onde estiver faltando
- Documentação inventário

**OUT:**
- Criar novas rotas API (admin atualmente usa Server Actions para a maior parte das mutações)
- Reescrever Server Actions existentes
- Mudanças em `/api/webhooks/*` (escopo F7.4)

## Contexto Técnico

- Hoje só existe `src/app/api/admin/turmas/[id]/resync-stripe/route.ts`. A maioria das mutações admin é feita via Server Actions dentro de `(admin)/admin/*/actions.ts` — esse padrão é OK e está fora do escopo desta story (ela cobre apenas `/api/admin/*`).
- `requireAdmin()` está em `src/lib/auth/helpers.ts`.
- Convenção de erro do projeto: `{ error: string }` no body, status apropriado.
