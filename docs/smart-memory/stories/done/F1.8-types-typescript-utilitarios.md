# Story F1.8: Geração de tipos TS + utilitários de banco

## Objetivo

Estabelecer fluxo de geração de tipos TypeScript e criar `/lib/supabase/` com clients tipados.

## Acceptance Criteria

- [x] AC1: Script `npm run db:types` em `package.json` que executa `supabase gen types typescript --project-id mksmmpfyqowuzjcchhkl > types/database.ts`
- [x] AC2: `types/database.ts` versionado e atualizado com schema atual
- [x] AC3: `src/lib/supabase/server.ts` exporta `createClient()` (Server Components/Actions) usando `@supabase/ssr` e cookies
- [x] AC4: `src/lib/supabase/client.ts` exporta `createClient()` (Client Components) via `createBrowserClient`
- [x] AC5: `src/lib/supabase/admin.ts` exporta `supabaseAdmin` usando `SUPABASE_SERVICE_ROLE_KEY`; marcado com `import 'server-only'`
- [x] AC6: Todos os clients tipados com `Database` de `types/database.ts`
- [x] AC7: Type aliases utilitários em `types/db.ts`: Profile, Course, Module, Lesson, Cohort, CohortMember, Payment, Coupon, etc — todos os enums derivados dos checks
