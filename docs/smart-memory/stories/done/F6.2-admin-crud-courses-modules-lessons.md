# Story F6.2: Admin — CRUD de courses, modules e lessons

## Objetivo

Interface admin para criar, editar e ordenar courses, modules e lessons; upload de vídeo via Vimeo API; editor com seletor de `content_format`; upload de materials por drag-and-drop.

## Acceptance Criteria

- [ ] AC1: `/admin/cursos` lista courses com busca, filtro published/draft, botão "Novo curso"
- [ ] AC2: `/admin/cursos/[id]/editar` edita curso (slug, title, description, cover_image_url) + lista modules com drag-and-drop para reordenar
- [ ] AC3: Toggle published/draft no curso
- [ ] AC4: CRUD de modules dentro do curso (criar, editar, deletar, reordenar via drag-and-drop)
- [ ] AC5: CRUD de lessons dentro de module:
  - Campos: slug, title, description, sort_order, kind (VIDEO/LIVE/IN_PERSON/CODE/READING), duration_seconds
  - Para `kind=VIDEO`: campo `video_provider` + `video_id` OU upload via Vimeo API
  - Seletor de `content_format` com 3 editores:
    - **MDX**: editor com preview lado a lado (Monaco editor)
    - **HTML**: textarea com preview sanitizado
    - **Markdown**: editor com syntax highlight ao vivo
- [ ] AC6: Upload de materials na lesson via drag-and-drop:
  - Sobe pro bucket `materials` em path `{lesson_id}/{material_id}.{ext}`
  - Cria registro em `materials` com kind detectado
- [ ] AC7: Botão "Preview como aluno" abre lesson no contexto de student
- [ ] AC8: Soft delete (set `deleted_at`) em vez de DELETE
- [ ] AC9: Reordenação atualiza `sort_order` em lote via Server Action transacional
- [ ] AC10: Validação Zod em todos os forms

## Escopo

**IN:**
- 3 níveis de CRUD com drag-and-drop
- Upload de vídeo via Vimeo API + opção de colar `video_id`
- 3 editores de conteúdo
- Upload de materials
- Preview como aluno

**OUT:**
- Versionamento de lesson (futuro)
- Editor TipTap WYSIWYG separado (parte do MDX/HTML editors)
- Workflow de aprovação (não há revisor além do admin)

## Contexto Técnico

- Drag-and-drop: `@dnd-kit/sortable`
- Upload Vimeo API: requisita upload URL, faz upload tus, salva `video_id` retornado
- Editor MDX: Monaco com preview server-side via Server Action que renderiza
- HTML editor: textarea simples + preview Server Component sanitizado (DOMPurify)
- Markdown editor: textarea + preview live com `react-markdown`
