# Story F9.8: Auditar VideoPlayer + flow de progresso (saveProgress, markComplete, edge cases)

## Objetivo
Garantir que o player de vídeo (Vimeo + YouTube) salva progresso de forma confiável, marca aulas como concluídas no threshold correto, e não regride o progresso quando o aluno tem múltiplas abas abertas.

## Acceptance Criteria
- [ ] AC1: Ao assistir uma aula em Vimeo até 95% do duration, a aula é marcada como `completed=true` automaticamente em `lesson_progress` (via `markLessonComplete`). Mesmo comportamento para YouTube. Reload da página mantém o estado completo. Verificar via UI (badge verde) e via query Supabase.
- [ ] AC2: `saveProgress` é idempotente e não-regressivo: se duas abas estão tocando o mesmo vídeo, a aba que parou em segundo 30 não pode sobrescrever o segundo 120 da outra aba. Validar lógica em `src/app/actions/progress.ts` (linhas 28-34: `update ... .lt('seconds_watched', seconds)`).
- [ ] AC3: Erros de rede em `saveProgress` ou `markLessonComplete` (`.catch(() => null)`) **não silenciam falhas críticas**: substituir por captura no Sentry ao menos para `markLessonComplete` (ações críticas). Errors de `saveProgress` podem ser silenciados (best-effort), mas devem ter rate-limit de log. Documentar no código a decisão.
- [ ] AC4: Edge cases cobertos por teste manual: (a) lesson com `video_provider=null` ou `video_id=null` mostra fallback "Vídeo não disponível" sem crash; (b) URL de YouTube inválida (sem 11 chars) mostra mensagem amigável; (c) trocar de provider entre Vimeo e YouTube na mesma sessão funciona (cleanup do player anterior é correto); (d) fechar aba durante playback dispara `saveProgress` no cleanup do useEffect.
- [ ] AC5: `MarkCompleteButton` (toggle manual) funciona em ambos os sentidos — marcar e desmarcar — sem race com o auto-complete do player. Se o player completa via 95% e o aluno depois desmarca manualmente, o estado fica desmarcado até nova ação.

## Contexto Técnico
- VideoPlayer usa `@vimeo/player` para Vimeo e iframe API para YouTube (lazy-loaded após click no thumbnail).
- `PROGRESS_INTERVAL_MS = 10_000` — chamada periódica de `saveProgress`.
- `COMPLETE_THRESHOLD = 0.95` — limiar para marcar como completo.
- `completedRef.current` previne dupla chamada de `markLessonComplete`.
- Server action `saveProgress` faz `upsert` + `update conditional` para evitar regressão.
- Após `markLessonComplete`, `triggerCertificateCheck` é fire-and-forget (catch via Sentry).
- Cuidado com cleanup de useEffect: tanto o Vimeo quanto YouTube têm cleanup que dispara `saveProgress` final.
