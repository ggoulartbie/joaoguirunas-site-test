# Story F1.2: Migration inicial — Identidade + Catálogo

## Objetivo

Criar a migration `initial_schema_identidade_catalogo.sql` com as tabelas de **identidade** (`profiles`) e **catálogo** (`courses`, `modules`, `lessons`, `materials`), incluindo triggers utilitários, sem RLS ainda (RLS vai em F1.5).

## Acceptance Criteria

- [x] AC1: Migration criada via `supabase migration new initial_schema_identidade_catalogo`
- [x] AC2: Tabela `profiles` com colunas exatas da PRD Seção 5.2 (id PK = auth.users.id, name, avatar_url, bio, role com check, stripe_customer_id, timestamps)
- [x] AC3: Tabelas `courses`, `modules`, `lessons`, `materials` com FKs e checks corretos do PRD
- [x] AC4: Função `set_updated_at()` criada e trigger `before update` aplicado em todas as tabelas com `updated_at`
- [x] AC5: Função `handle_new_user()` + trigger `on_auth_user_created` em `auth.users` criando `profiles` automaticamente com role `STUDENT`
- [x] AC6: Constraints únicas: `courses.slug`, `(modules.course_id, modules.slug)`, `(lessons.module_id, lessons.slug)`
- [x] AC7: `supabase db push` aplica sem erros e `supabase db diff` retorna vazio depois
- [x] AC8: Tipos TS regenerados via `supabase gen types typescript --project-id mksmmpfyqowuzjcchhkl > types/database.ts`

## Escopo

**IN:**
- 5 tabelas: `profiles`, `courses`, `modules`, `lessons`, `materials`
- Triggers `set_updated_at` e `handle_new_user`
- Soft delete (`deleted_at`) em courses, modules, lessons
- Geração de tipos TS

**OUT:**
- Tabelas de cohorts (F1.3)
- Tabelas de pagamento, progresso, fórum, certificados (F1.4)
- Função `has_access()` (F1.5)
- RLS (F1.5)
