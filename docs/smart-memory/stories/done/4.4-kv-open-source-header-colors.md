# Story 4.4: KV Open-Source — header editorial + unificação do sistema de cores

## Objetivo
A /open-source é a página que mais diverge do KV: começa sem header editorial, não tem assinatura da GrowthSales.ai, e usa 5 cores saturadas que quebram o sistema bone+ember. Esta story adiciona o header e migra o sistema de cores para hairline+opacidade.

## Acceptance Criteria
- [x] AC1: **Header editorial** adicionado no topo de `OpenSourceClient`: H1 "Open Source" em Fraunces via CSS variable, `text-5xl sm:text-7xl`, `tracking-tight`, `color: white`
- [x] AC2: **Sub-headline** abaixo do H1: `"Ferramentas reais para Claude Code — curadas por João Guirunas e a comunidade GrowthSales.ai"`, `text-sm sm:text-lg`, `max-w-2xl`
- [x] AC3: **Mono eyebrow** acima do H1: `"40+ Recursos · Gratuitos · Open Source"` em `--font-mono`, 11px, ember, `letter-spacing: 0.16em`
- [x] AC4: **Sistema de cores das categories** migrado — `bg-transparent` em todas as 5 categorias, `border: 1px solid {cor}/25`
- [x] AC5: **Filtro "Todos"** adicionado como primeiro item no array `categories` (id: 'all')
- [x] AC6: Header com `GrowthWatermark size={500}` posicionado no canto direito do bloco header
- [x] AC7: Mobile: header responsivo com `text-5xl sm:text-7xl`, sub-headline `text-sm sm:text-lg`

## Escopo

**IN:**
- `src/app/open-source/open-source-client.tsx` — header + sistema de cores
- Nenhum card de skill é removido ou reordenado

**OUT:**
- Dados dos skills (títulos, descrições, links, autores)
- Ícones SVG dos cards
- Lógica de filtro (só estilo dos botões de filtro)

## Contexto Técnico

**OpenSourceClient** é um client component (`'use client'`). Fraunces via CSS variable (`--font-display-serif`) funciona via `className` com `font-[family-name:var(--font-display-serif)]` — funciona em client components desde que a fonte seja carregada no layout.tsx (story 4.2).

**Categories atual**:
```tsx
{ id: 'skills', label: 'Skills', color: 'bg-[#22C55E]/20 text-[#22C55E] border-[#22C55E]/30' },
```

**Categories target** (manter cor do texto, remover bg):
```tsx
{ id: 'skills', label: 'Skills', activeColor: 'text-[#22C55E] border-[#22C55E]/40', inactiveColor: 'text-[#22C55E]/50 border-white/10' },
```

**KV Rule**: *"Nunca duas cores saturadas adjacentes."* Cards com bg colorido violam; badges só com border são neutros.

**Header layout**:
```
[mono eyebrow]
[H1 grande]
[sub-headline]
[filtros]
[grid de cards]
```

Espaçamento: `pt-24 pb-16` no header (espaço generoso — KV: "whitespace as luxury").
