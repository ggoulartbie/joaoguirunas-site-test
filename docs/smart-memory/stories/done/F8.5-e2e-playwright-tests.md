# Story F8.5: Testes E2E com Playwright

## Objetivo

Suite mínima de testes E2E cobrindo os fluxos críticos: cadastro→checkout→aula→progresso, bloqueio de conteúdo, idempotência de webhook.

## Acceptance Criteria

- [ ] AC1: Playwright instalado e configurado (`playwright.config.ts`)
- [ ] AC2: Test 1 — Fluxo de compra:
  - Cadastro de novo aluno
  - Acessa `/turmas/online-padrao`
  - Clica em "Quero entrar"
  - Stripe Checkout em modo test (cartão `4242...`)
  - Redireciona pra /dashboard com cohort ativa
  - Acessa /curso/aiox-fundamentos/aula/... e VideoPlayer carrega
  - Salva progresso e ao recarregar mostra retomada
- [ ] AC3: Test 2 — Bloqueio de conteúdo:
  - Aluno em cohort online tenta acessar aula da cohort mentoria
  - Recebe `<LockedContent />` com CTA correto
  - DOM **não** contém `video_id`
- [ ] AC4: Test 3 — Webhook idempotência:
  - Disparar `checkout.session.completed` 2x via Stripe CLI ou mock
  - Verificar que apenas 1 `payment` foi criado
  - Verificar que `cohort_members` não duplica
- [ ] AC5: Test 4 — Cross extension:
  - Aluno com cohort_member em mentoria
  - Compra cohort online
  - Verificar que prazo da mentoria foi estendido em N dias (`days_granted`)
- [ ] AC6: Tests rodam em CI (GitHub Actions) em PRs
- [ ] AC7: Banco de teste isolado (Supabase branch ou seed dedicado)

## Escopo

**IN:**
- 4 testes críticos
- CI integration

**OUT:**
- Testes unitários (cobertura por library — não no MVP)
- Testes visuais (post-MVP)

## Contexto Técnico

- Stripe modo test com cartões de teste oficiais
- Vimeo: usar vídeo de teste curto ou stub
- Banco: usar migration de seed dedicado (`supabase db reset` antes de cada suite)
