# Story F10.5: Verificar + completar webhook Stripe → cria conta automaticamente

## Objetivo
Garantir que quando um aluno compra o Curso Online via Stripe, o webhook cria automaticamente a conta Supabase, o perfil e a matrícula no cohort correcto, e envia o email de boas-vindas com acesso.

## Acceptance Criteria

- [x] AC1: Evento `checkout.session.completed` — extrai `customer_details.email`, `customer_details.name`, e metadata `cohort_slug`
- [x] AC2: `findOrCreateUser()` verifica se user existe por email (listUsers) — se sim, só cria member; se não, cria user
- [x] AC3: `supabaseAdmin.auth.admin.createUser()` com email, nome, `email_confirm: true`
- [x] AC4: Cria `cohort_members` status ACTIVE, `expires_at` via `cohort.access_duration_days`. Upsert idempotente (se já existe, faz UPDATE)
- [x] AC5: Gera magic link via `generateLink({ type: 'magiclink', email })` → envia `WelcomeInviteEmail` com o link
- [x] AC6: `StripeAdapter.createCheckoutSession` já passava `cohort_slug` no metadata — verificado e mantido; `userId` agora opcional
- [x] AC7: Idempotência via `webhook_events` (UNIQUE em `stripe_event_id`) — já existia; upsert no member garante sem duplicata mesmo se webhook_events falhar antes do insert
- [ ] AC8: Teste manual com Stripe CLI — requer env vars `STRIPE_*` configuradas

## Notas de implementação

- Dois fluxos no `handleCheckoutCompleted`: autenticado (`user_id` no metadata) e público (sem `user_id`, usa `cohort_slug`)
- Fluxo público implementado em `handlePublicCheckoutCompleted()` separado para clareza
- `createPublicCheckoutSession()` em `src/app/actions/checkoutPublic.ts` — para LP pública `/curso-online` usar sem requerer login
- `CreateCheckoutParams.userId` e `customerEmail` agora opcionais na interface
