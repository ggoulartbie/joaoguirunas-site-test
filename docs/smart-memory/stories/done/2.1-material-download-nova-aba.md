# Story 2.1: Download de material abre em nova aba

## Objetivo
Garantir que ao clicar em "Baixar" / "Abrir" na aba **Materiais** da página de aula do aluno, o navegador NÃO substitua a aba atual — disparar download de arquivos sem navegação e abrir links externos em nova aba.

## Acceptance Criteria

- [ ] **AC1**: Ao clicar em material com `kind != 'LINK'` (PDF, ZIP, IMAGE, OTHER), o arquivo é baixado/aberto **sem** navegar a aba atual da página de aula. A URL da aba do aluno permanece em `/academy/curso/[slug]/aula/[lesson-slug]`.
- [ ] **AC2**: Ao clicar em material com `kind = 'LINK'`, o destino abre em **nova aba** (`target="_blank"`) com `rel="noopener noreferrer"`.
- [ ] **AC3**: O atributo `download` do `<a>` programático recebe o `material.title` apenas para arquivos não-LINK; o nome sugerido aparece corretamente no diálogo de save do browser quando o `Content-Disposition` da signed URL não força nome.
- [ ] **AC4**: Estado de loading (`Preparando...`) e estado de erro continuam funcionando — sem regressão visual no botão.
- [ ] **AC5**: Validação manual em Chromium e Firefox: clicar em PDF não muda a URL da aba; clicar em LINK abre nova aba; clicar em ZIP dispara download.

## Escopo

**IN:**
- Editar `src/components/student/MaterialsList.tsx` na função `handleDownload`: para `kind != 'LINK'`, adicionar também `a.target = '_blank'` + `a.rel = 'noopener noreferrer'` (defesa contra navegação) **OU** fazer fetch + Blob + `URL.createObjectURL` (forçar download local).
- Solução recomendada: `target="_blank" + rel + download` no anchor — funciona em todos os browsers modernos. O browser respeita `download` quando same-origin OU quando a signed URL retorna `Content-Disposition: attachment`. Como o Supabase Storage de buckets privados serve com `Content-Disposition: inline` por default para PDFs/imagens (que abrem inline no browser), `target="_blank"` evita o pior caso de "perdi o estado da aula".

**OUT:**
- Alterar a server action `downloadMaterialAction` ou `getMaterialSignedUrl` (lógica de signed URL inalterada).
- Mudar TTL da signed URL ou storage policy.
- Refatorar visual do botão / KV / iconografia.
- Touch no admin (escopo da Story 2.2).
- Telemetria de cliques em materiais.
- Forçar `Content-Disposition: attachment` via parâmetro `download` na signed URL do Supabase (otimização opcional, fica para follow-up).

## Contexto Técnico

**Arquivo afetado:**
- `src/components/student/MaterialsList.tsx` (linhas 31-56) — função `handleDownload`.

**Diagnóstico do bug:**
- Hoje, linha 42-47: para arquivos (`kind != 'LINK'`) seta apenas `a.download = material.title`. **Sem** `target` nem `rel`.
- Para uma signed URL do Supabase Storage que retorna `Content-Disposition: inline` (default em PDFs/imagens), o atributo `download` é ignorado em URLs cross-origin sem CORS adequado, e o browser navega na aba atual exibindo o PDF inline. Resultado: o aluno perde a página da aula (com vídeo, comentários, progresso).
- Já existe `target="_blank"` no branch LINK (linha 45-46) — basta extender para o branch de arquivo.

**Snippet alvo (referência, não copiar literalmente — dev valida o melhor approach):**
```tsx
const a = document.createElement('a')
a.href = url
a.target = '_blank'                 // NOVO — em ambos os branches
a.rel = 'noopener noreferrer'       // NOVO — em ambos os branches
if (material.kind !== 'LINK') {
  a.download = material.title
}
document.body.appendChild(a)
a.click()
document.body.removeChild(a)
```

**Tracking signed URL TTL**: signed URL do bucket `materials` é 5min (migration `20260506022229_storage_buckets_policies.sql:30`). `target="_blank"` mantém a aba do aluno viva mesmo se o usuário só ler o PDF na nova aba.
