# Story F6.1: Admin Dashboard

## Objetivo

Página `/admin` com métricas operacionais: receita, alunos ativos, conclusão, moderação pendente, pagamentos recusados, matrículas expirando.

## Acceptance Criteria

- [ ] AC1: `app/(admin)/admin/page.tsx` (RBAC ADMIN)
- [ ] AC2: Card "Receita total" e "Receita do mês" (soma de `payments.amount_cents` com `status=APPROVED`)
- [ ] AC3: Card "Alunos ativos" total e por cohort (count `cohort_members` ACTIVE)
- [ ] AC4: Card "Conclusão média" (% de lessons completadas / total esperado por cohort)
- [ ] AC5: Card "Moderação" — count de comments e forum_threads novos nas últimas 24h
- [ ] AC6: Card "Pagamentos recusados últimas 24h"
- [ ] AC7: Tabela "Matrículas expirando nos próximos 7 dias" com nome, cohort, expires_at, ação "Estender manualmente"
- [ ] AC8: Gráfico simples (line chart) de receita últimos 30 dias
- [ ] AC9: Layout admin compartilhado: sidebar com nav (Cursos, Turmas, Usuários, Pagamentos, Moderação, Fórum)

## Escopo

**IN:**
- Dashboard
- Layout admin shell
- Queries agregadas

**OUT:**
- Drill-down em cada métrica (cards são read-only neste momento)
- Exportação CSV (na Fase 7 de pagamentos)

## Contexto Técnico

- Queries pesadas: usar `materialized view` ou cache de 5min via `unstable_cache`
- Charts: recharts ou tremor
