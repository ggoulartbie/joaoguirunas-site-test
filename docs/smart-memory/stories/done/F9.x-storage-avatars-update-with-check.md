# Story F9.x: Fix storage — avatars UPDATE sem with check

## Contexto

Identificado pelo QA durante F1.6 como concern não-bloqueante. A policy de UPDATE no bucket `avatars` não tinha `WITH CHECK`, e havia policies duplicadas com nomes diferentes das migrations 022229 e 100000.

## Acceptance Criteria

- [x] AC1: Policies duplicadas de avatars (origens: migrations 022229 e 100000) removidas com `DROP POLICY IF EXISTS`
- [x] AC2: Policy `avatars: update pelo próprio usuário` recriada com `WITH CHECK` que valida path de destino via `auth.uid()`
- [x] AC3: Migration aplicada via `supabase db push`
- [x] AC4: Documentação em `storage.md` atualizada

## Correção aplicada

```sql
drop policy if exists "avatars: update pelo próprio usuário" on storage.objects;
drop policy if exists "avatars_update_own" on storage.objects;
-- (e demais duplicatas)

create policy "avatars: update pelo próprio usuário"
  on storage.objects for update
  to authenticated
  using (
    bucket_id = 'avatars'
    and auth.uid()::text = (storage.foldername(name))[1]
  )
  with check (
    bucket_id = 'avatars'
    and auth.uid()::text = (storage.foldername(name))[1]
  );
```
