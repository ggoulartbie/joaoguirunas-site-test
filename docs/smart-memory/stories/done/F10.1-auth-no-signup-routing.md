# Story F10.1: Auth — desactivar signup público + routing pós-login

## Objetivo
Remover a opção de criar conta das páginas de auth e garantir que após login todos os utilizadores (incluindo ADMIN) vão sempre para `/academy/aluno` por defeito.

## Acceptance Criteria

- [x] AC1: Supabase Auth — "Disable Sign Ups" activado via painel (config manual documentada em `docs/smart-memory/agents/data-engineer/auth-config.md` secção 1)
- [x] AC2: Página `/academy/login` — removido parágrafo "Não tem conta? Cadastre-se" e link para `/academy/cadastro`; `/academy/cadastro` também removida de PUBLIC_PATHS do middleware
- [x] AC3: Após login bem-sucedido, `LoginForm.tsx` redireciona SEMPRE para `/academy/aluno` (sem depender de `?next=`); `auth/callback/route.ts` usa `/academy/aluno` como default
- [x] AC4: `/academy` (raiz) — middleware intercpta e redireciona: autenticado → `/academy/aluno`; não autenticado → `/academy/login`; também existe `app/(academy)/academy/page.tsx` com mesma lógica como fallback
- [x] AC5: Fluxo de magic link (convite/redefinição) continua funcional — `auth/callback` e `redefinir-senha` não foram alterados além do default redirect
- [x] AC6: `/academy/aluno` adicionado ao matcher do middleware — sem login redireciona para `/academy/login`; após login chega a `/academy/aluno` que redireciona para `/academy/dashboard`

## Escopo
**IN:** middleware updates, remoção de UI de signup, redirect logic
**OUT:** criação do badge admin (F10.2), criação manual de alunos (F10.4)
