# Story F2.3: Middleware Next.js + helpers RBAC

## Objetivo

Implementar middleware que valida sessão Supabase em rotas protegidas e helpers `getCurrentUser`, `requireUser`, `requireRole`, `requireAdmin` para uso em Server Components e Server Actions.

## Acceptance Criteria

- [x] AC1: `middleware.ts` na raiz do projeto — atualiza sessão via `@supabase/ssr`, protege `/academy/dashboard/*`, `/academy/meus-cursos/*`, `/academy/curso/*`, `/academy/forum/*`, `/academy/agenda/*`, `/academy/certificados/*`, `/academy/perfil/*` com redirect pra `/academy/login?next=...`; protege `/academy/admin/*` com check de role ADMIN
- [x] AC2: `src/lib/auth/helpers.ts` exporta: `getCurrentUser()` → `Profile | null` (cached via React `cache()`), `requireUser()` → `Profile` ou redirect, `requireRole(roles: Role[])` → `Profile` ou redirect 403, `requireAdmin()` → `Profile` ADMIN ou redirect 403
- [x] AC3: Helpers buscam profile completo do Postgres (`select('*')`) — não dependem só de metadata Supabase auth
- [x] AC4: Acessar `/academy/admin` sem login redireciona para `/academy/login`
- [x] AC5: Acessar `/academy/admin` como STUDENT redireciona para `/academy/403` (page customizada existe)
- [x] AC6: Acessar `/academy/dashboard` sem login redireciona para `/academy/login?next=/academy/dashboard`
- [x] AC7: `redirectTo`/`next` query param preservado no redirect — LoginForm lê `?next=` e redireciona após login
- [x] AC8: `/api/webhooks/*` excluído do matcher de auth (está em PUBLIC_PATHS)

## Escopo

**IN:**
- `middleware.ts` com matcher correto
- Helpers em `lib/auth/helpers.ts`
- Página de erro 403
- `getCurrentUser()` cached por request via React `cache()`

**OUT:**
- Helpers de banco para `has_access` (Fase 3)
- UI de "logado como" no header (Fase 8)
