# Story 6.1: Upgrade texturas dos planetas para 4K

## Objetivo
Substituir as texturas SD dos 4 planetas por versões de alta resolução, usando jupiter-4k.jpg já disponível e aplicando a mesma qualidade nos demais planetas (Mars, Venus, Neptune).

## Contexto
O SolarSystemScene.tsx usa 4 planetas:
- Dev squad → Jupiter (`/textures/planets/jupiter.jpg` 240KB → já existe `jupiter-4k.jpg` 2.9MB)
- Sites squad → Mars (`/textures/planets/mars.jpg` 320KB — precisa upgrade)
- Social squad → Venus (`/textures/planets/venus.jpg` 1.5MB — já HD)
- Traffic squad → Neptune (`/textures/planets/neptune.jpg` 236KB — menor de todos, precisa upgrade)

Existe uma página de teste em `/test-planet` que pode ser usada para validar.

## Acceptance Criteria
- [x] AC1: `jupiter-4k.jpg` é usado no lugar de `jupiter.jpg` em SolarSystemScene.tsx
- [ ] AC2: Mars e Neptune têm texturas de maior resolução — PENDENTE (solarsystemscope.com retornou 404 para /download/4k_mars.jpg e /download/4k_neptune.jpg; texturas SD originais mantidas)
- [x] AC3: Starfield usa `stars-milky-way.jpg` via MilkyWayBackground component (EquirectangularReflectionMapping no scene.background)
- [x] AC4: Texturas carregam sem quebrar o build (`pnpm build` passou)
- [x] AC5: jupiter-4k.jpg = 2.9MB (dentro do limite 4MB); stars-milky-way.jpg = 1.8MB
