# Story F3.3: Página da aula + has_access + LockedContent

## Objetivo

Página `/curso/[slug]/aula/[lesson-slug]` como Server Component que valida acesso via `has_access()` e renderiza player+conteúdo OU `<LockedContent />`.

## Acceptance Criteria

- [x] AC1: Page é Server Component
- [x] AC2: Carrega lesson + module + course pelo slug; 404 via `notFound()` se não existir
- [x] AC3: Chama `has_access(user_id, lesson_id)` RPC no servidor
- [x] AC4: `false`: renderiza `LockedContent` com cadeado e CTA. `video_id` nunca enviado ao DOM sem acesso
- [x] AC5: `true`: renderiza `VideoPlayer` + conteúdo da aula + tabs (Materiais, Comentários) + CourseSidebar
- [x] AC6: `CourseSidebar` mostra módulos do curso com estado de completado/bloqueado
- [x] AC7: Navegação anterior/próxima presente (prev/next cards no bottom)
- [x] AC8: `generateMetadata` dinâmica com título da aula
- [ ] AC8 (noindex): `robots: { index: false }` ausente na metadata — páginas de aula estão indexáveis; considerar adicionar se necessário para SEO
- [x] AC9: Mobile responsive (MobileLessonDrawer para CourseSidebar em mobile)
