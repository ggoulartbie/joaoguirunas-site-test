# Story F2.4: Emails transacionais — Resend + React Email

## Objetivo

Configurar Resend como provedor de emails e criar templates iniciais (boas-vindas, recuperação de senha) usando React Email.

## Acceptance Criteria

- [x] AC1: Domínio verificado no Resend documentado em `email-templates.md` com instruções SPF/DKIM
- [x] AC2: `RESEND_API_KEY` e `RESEND_FROM_EMAIL` em `.env.example`
- [x] AC3: `src/lib/email/send.ts` com funções tipadas por template (sendWelcomeEmail, sendPasswordResetEmail, etc.) usando Resend SDK
- [x] AC4: `emails/templates/WelcomeEmail.tsx` — email de boas-vindas pós-cadastro com design KV
- [x] AC5: `emails/templates/PasswordResetEmail.tsx` — email de recuperação de senha
- [x] AC6: Templates usam paleta KV (#050507 bg, #ff3a0e ember, #f1f1f3 bone), Georgia italic para headings, Courier New para labels
- [x] AC7: React Email gera fallback texto puro automaticamente via `render()`
- [x] AC8: `auth/callback/route.ts` dispara `sendWelcomeEmail` no primeiro login pós-confirmação (detecta conta nova por `created_at < 5min`)
- [x] AC9: Documentação em `docs/smart-memory/agents/marketing/email-templates.md` com lista completa de templates e triggers
- [x] AC10: Resend SDK com lazy init (key lida em runtime, não em build) — compatível com Vercel preview sem key

## Escopo

**IN:**
- Setup Resend
- 11 templates (welcome, reset, payment, cohort, cron triggers, certificate)
- Helper `send.ts` com funções por template
- Disparo do welcome no callback OAuth/email

**OUT:**
- Templates de pagamento activos (aguardam Fase 7 para ligação ao webhook Stripe)
- Email de expiração próxima activo (aguarda Fase 7 cron)
