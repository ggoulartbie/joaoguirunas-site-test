# Story F9.16: Auditar components student críticos (VideoPlayer, Materials, Comments)

## Objetivo
Auditar internamente os components de maior risco em `src/components/student/*`: `VideoPlayer`, `MaterialsList`, `CommentsSection`, `LockedContent`, `MarkCompleteButton`, `NotificationBell`. Foco em XSS no render de conteúdo, leak de `video_id`, TTL de signed URLs, integridade de chamadas de Server Action.

## Acceptance Criteria
- [x] AC1: `VideoPlayer.tsx` — videoId não exposto sem hasAccess (gate no server), throttle 10s, providers apenas VIMEO/YouTube (sem URL arbitrária).
- [x] AC2: `CommentsSection.tsx` — conteúdo renderizado como texto React (escapado), sem dangerouslySetInnerHTML. XSS seguro.
- [x] AC3: `MaterialsList.tsx` — signed URL TTL = 5min, gerada server-side, acesso validado via has_access RPC antes de gerar URL.
- [x] AC4: `MarkCompleteButton.tsx` — toggleLessonComplete resolve user via requireUser() server-side, upsert com user.id da sessão (anti-IDOR).
- [x] AC5: `LockedContent.tsx` — não expõe video_id, description, materials. Apenas cohortName e cohortDescription.
- [x] AC6: `NotificationBell.tsx` + `OnboardingWizard.tsx` — sem polling, sem auth token no client, usam Server Actions para mutação.
- [x] AC7: `renderContent()` — HTML sanitizado via DOMPurify com allowlist, MDX via serializeMDX. Sem XSS.
- [x] AC8: Nenhum bug P0/P1 encontrado. mock-data.ts não importado em produção.
