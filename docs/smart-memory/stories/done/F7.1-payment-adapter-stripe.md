# Story F7.1: Payment adapter + Stripe implementation

## Objetivo

Criar abstração de pagamento (`/lib/payment/`) com interface neutra e implementação Stripe, permitindo trocar PSP futuramente.

## Acceptance Criteria

- [ ] AC1: `lib/payment/interface.ts` define `PaymentProvider` com métodos: `createCheckoutSession(args)`, `createCustomer(args)`, `createCoupon(args)`, `createProduct(args)`, `createPrice(args)`, `archivePrice(id)`, `refundPayment(args)`, `verifyWebhookSignature(args)`
- [ ] AC2: `lib/payment/stripe.ts` implementa via SDK oficial `stripe`
- [ ] AC3: Singleton em `lib/payment/index.ts` baseado em env (`PAYMENT_PROVIDER=stripe`)
- [ ] AC4: Wrapper `import 'server-only'` no client Stripe (chave secreta nunca no client)
- [ ] AC5: Tipos TS exportados (CheckoutSessionInput, etc) para uso em outras stories
- [ ] AC6: Variáveis de ambiente: `STRIPE_SECRET_KEY`, `STRIPE_PUBLISHABLE_KEY`, `STRIPE_WEBHOOK_SECRET` documentadas

## Escopo

**IN:**
- Interface neutra
- Stripe implementation
- Singleton/factory

**OUT:**
- Sync cohort↔Stripe (F7.2)
- Webhook (F7.3)
- Auto renewal (F7.4)

## Contexto Técnico

- ADR-001 D2.8: Stripe Checkout (hosted) — sem PCI scope
- API version pinned em produção (evita breaking changes silenciosos)
- Erros do Stripe têm `error.type` (StripeCardError, StripeAPIError, etc) — mapear para erros do app
