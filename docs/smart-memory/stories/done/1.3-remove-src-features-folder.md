# Story 1.3: Remover src/features/ (pasta inutilizada)

## Objetivo
Remover a pasta `src/features/` contendo apenas `.gitkeep` — o projeto adota de fato o padrão de co-location em `src/app/`, então manter a pasta vazia indica uma convenção que não existe e confunde novos contribuidores.

## Acceptance Criteria
- [x] AC1: Pasta `src/features/` removida por completo (incluindo `.gitkeep`).
- [x] AC2: `tsconfig.json` não contém alias para `@/features/*`.
- [x] AC3: `grep -r "@/features\|src/features\|from.*features/" src/` retorna zero resultados.
- [x] AC4: `tsc --noEmit` conclui sem erros.
- [ ] AC5: Documentação `docs/smart-memory/project/architecture.md` e `conventions.md` atualizadas.
- [ ] AC6: Item 4 de "Pontos de atenção" em `overview.md` removido.

## Escopo

**IN:**
- `rm -rf src/features/`.
- Verificar e limpar aliases de `tsconfig.json` se apontarem para `features/`.
- Atualizar docs em `smart-memory/project/` (architecture, conventions, overview).

**OUT:**
- Migrar código existente pra um padrão `features/` (não é a intenção — o padrão adotado é co-location em `src/app/`).
- Refatoração de estrutura de pastas além desta remoção.
- Criação de nova convenção documentada separando "componentes compartilhados" vs "features" (fora de escopo; seria outra story se decidirmos formalizar).

## Contexto Técnico

**Evidência:**
- `ls src/features/` → contém apenas `.gitkeep` (arquivo vazio, 0 bytes, criado em 04-04-2025).
- `docs/smart-memory/project/overview.md:76` item 4: "src/features/ existe com .gitkeep mas é inutilizada — padrão real é co-location em src/app/".
- `docs/smart-memory/project/modules.md:234` confirma: "Pasta src/features/: vazia (apenas .gitkeep). Indica intenção de organizar por feature mas ainda não adotada".

**Decisão arquitetural implícita:** o padrão real adotado em todas as rotas é co-location — ex.: `src/app/mentoria/*.tsx` mantém 12 componentes específicos da rota dentro da própria pasta da rota. Esse é o padrão documentado em [[../../project/architecture]] §Co-location-Rule. Manter `src/features/` dá falsa impressão de estrutura alternativa válida.

**Risco:** baixo. Pasta não é referenciada em nenhum arquivo. Se no futuro houver decisão de adotar arquitetura feature-based, isso deve ser formalizado via ADR antes de recriar a pasta.
