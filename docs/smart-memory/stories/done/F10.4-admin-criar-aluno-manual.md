# Story F10.4: Admin — criar aluno manual + envio de convite

## Objetivo
Permitir que o João crie contas de alunos manualmente no painel admin (para alunos da Mentoria ou casos excepcionais), com envio automático de email de convite com link de acesso.

## Acceptance Criteria

- [ ] AC1: Em `/academy/admin/alunos` — botão "Criar aluno" abre modal/form com campos: Nome completo, Email, Cohort (dropdown com cohorts activos), Role (STUDENT/MENTOR)
- [ ] AC2: Action `createStudentManually(data)` usa `supabaseAdmin.auth.admin.createUser()` com `email_confirm: true` e `send_email: false` (enviamos o email nós)
- [ ] AC3: Após criar user, cria registo em `profiles` (nome, role) e `cohort_members` (cohort seleccionado, status ACTIVE)
- [ ] AC4: Email de convite enviado via Resend com template "WelcomeInvite" — inclui link de magic link ou link para definir senha (`supabaseAdmin.auth.admin.generateLink()` type `magiclink`)
- [ ] AC5: Template email "WelcomeInvite" criado em `emails/templates/` com: boas-vindas, nome do aluno, nome do cohort, botão "Activar minha conta"
- [ ] AC6: Feedback no admin: toast de sucesso com "Aluno criado e convite enviado para {email}" ou erro detalhado
- [ ] AC7: Aluno criado aparece imediatamente na listagem de UsersClient sem refresh manual

## Escopo
**IN:** action de criação, template email, UI modal no admin
**OUT:** gestão de pagamentos deste aluno (admin já tem essa secção), fluxo Stripe (é manual, não passa pelo checkout)
