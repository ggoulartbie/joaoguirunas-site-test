# Story F3.5: Dashboard + /meus-cursos + /curso/[slug]

## Objetivo

Páginas de entrada do aluno: dashboard com saudação e progresso, listagem de cursos liberados (união das cohorts ativas) e página de curso com módulos/aulas.

## Acceptance Criteria

- [x] AC1: `app/(student)/dashboard/page.tsx` mostra:
  - Saudação + nome do aluno
  - Card "Continue de onde parou" (última lesson com seconds_watched > 0 e não completada)
  - Próximo encontro ao vivo (se aplicável)
  - Cards das cohorts ativas com % de progresso (lessons completadas / total liberadas)
  - Aviso "Sua matrícula em X expira em N dias" se `expires_at` < 30 dias
- [x] AC2: `app/(student)/meus-cursos/page.tsx` lista cursos da união das cohorts ativas; cursos parcialmente liberados indicam "Módulos X/Y disponíveis" com `isPartialAccess`
- [x] AC3: `app/(student)/curso/[slug]/page.tsx` mostra módulos do curso; módulos liberados navegáveis; bloqueados com cadeado
- [x] AC4: Lógica de "qual cohort libera": JOIN de `cohort_members` (ACTIVE) com `cohort_courses`; respeita `included_module_ids`
- [x] AC5: Estados vazios em todas as páginas (nenhuma cohort ativa, nenhuma lesson iniciada)
- [x] AC6: Mobile responsive (grid responsivo sm/lg)
- [x] AC7: Queries usam supabaseAdmin com índices existentes
