# Story F8.1: Notifications in-app + onboarding

## Objetivo

Sininho de notifications no header do aluno (lê tabela `notifications`) e wizard de onboarding no primeiro login.

## Acceptance Criteria

- [x] AC1: `<NotificationBell>` no header com badge de count não lidas — integrado em `StudentTopBar`
- [x] AC2: Dropdown com últimas 20 notifications; `markAllNotificationsRead` disponível
- [x] AC3: Click em notification marca como lida (`read_at=now()`) via `markNotificationRead` server action e navega para `action_url`
- [x] AC4: Notifications criadas por `checkAndIssueCertificate` (certificate pronto), cron diário (matrícula expirando), `material-actions.ts` (novo material)
- [x] AC5: `OnboardingWizard` no primeiro login (detectado por `!profile.name && !profile.bio`):
  - Step 1: boas-vindas
  - Step 2: tour das áreas
  - Step 3: preencher nome + bio (Skip permitido)
- [ ] AC6: Realtime via Supabase — não implementado (nice-to-have, out of scope MVP)
