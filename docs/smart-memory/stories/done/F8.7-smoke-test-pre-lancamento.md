# Story F8.7: Smoke test pré-lançamento

## Objetivo

Checklist manual executado em ambiente de produção (com dados reais, exceto compras reais com cartão simbólico) confirmando que a plataforma está pronta para receber o primeiro aluno real.

## Acceptance Criteria

- [ ] AC1: Checklist documentado em `docs/smart-memory/runbooks/go-live-checklist.md` (cobrindo todos os pontos abaixo)
- [ ] AC2: **Auth**: cadastro novo recebe email; login funciona; reset de senha completa; Google OAuth (se ativo) autentica
- [ ] AC3: **Conteúdo bloqueado**: usuário recém-cadastrado **não** vê nenhum conteúdo de cursos; LP `/turmas/[slug]` renderiza com preço correto
- [ ] AC4: **Compra real (smoke)**: 1 admin compra real (R$ baixo simbólico, ou via cupom 100%) em modo produção:
  - Stripe Checkout abre com Price correto
  - Pagamento aprovado
  - Webhook em produção cria membership
  - Aluno acessa `/dashboard` e vê cohort ativa em < 2 min
  - Acessa lesson, player carrega, progress salva
- [ ] AC5: **Cross extension**: comprar a cohort source aplica `days_granted` na target (validar em SQL e UI)
- [ ] AC6: **Refund**: admin faz refund via `/admin/pagamentos`; webhook processa; status `payments.status='REFUNDED'`
- [ ] AC7: **Emails reais chegam**: welcome, access-granted, payment-receipt
- [ ] AC8: **Performance**: Lighthouse mobile em /turmas/[slug] e /dashboard com Performance > 80, Best Practices > 90
- [ ] AC9: **RLS**: rodar query de auditoria validando todas as tabelas com `rowsecurity=true`
- [ ] AC10: **Sentry capturando**: forçar 1 erro controlado em /admin (admin-only) e verificar que aparece com tags corretas
- [ ] AC11: **Cron diário**: forçar manualmente via `curl -H "Authorization: Bearer $CRON_SECRET" https://{prod}/api/cron/daily` e verificar logs
- [ ] AC12: **Backup**: confirmar backup do dia no painel Supabase
- [ ] AC13: Documentar o resultado em `docs/smart-memory/runbooks/launch-{YYYY-MM-DD}.md` com prints/saídas

## Escopo

**IN:**
- Checklist documentado
- Execução em produção
- Documentação do resultado

**OUT:**
- Testes automatizados em produção (caso vire necessidade, virar synthetic monitoring post-MVP)

## Contexto Técnico

- Aceite final do PRD (Seção 8 Fase 8): "Primeiro aluno real consegue completar fluxo ponta-a-ponta" — esse smoke test é o gating
- Métricas-alvo (Seção 2.3): tempo até primeiro acesso < 2min, uptime > 99,5%, response < 800ms
- Bloqueio de go-live: qualquer item NO-GO deve virar issue antes do anúncio público
