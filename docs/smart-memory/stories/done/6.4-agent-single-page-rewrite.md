# Story 6.4: Rewrite da single page do agente com conteúdo rico

## Objetivo
Reescrever `src/app/agentes/[squad]/[agent]/page.tsx` para exibir informações ricas do agente: foto real (PNG já existe), raça alienígena com lore, tagline, abilities, squadRole, authorities, e identidade.

## Contexto
A single page está em `src/app/agentes/[squad]/[agent]/page.tsx`.
Esta story depende da Story 6.2 (abilities, squadRole, codenames no JSON).

**Problemas atuais:**
1. Hero usa `initials` (letras gigantes) — mas existe `/agentes/{slug}.png` para todos os 37 agentes
2. Não mostra `tagline` do agente
3. Não mostra `abilities` (nova da 6.2)
4. Não mostra `squadRole` (nova da 6.2)
5. `race` e `archetype` aparecem em uma linha sutil — merecem mais destaque
6. Seção "O que faz" é só a description — precisa ser mais rica

## Nova estrutura de página proposta

### HERO (acima do fold)
- Coluna esquerda: foto real `/agentes/{slug}.png` (usar next/image, object-cover object-top)
- Coluna direita:
  - Squad badge + ID
  - Codename (h1 grande)
  - Title (italic, subdued)
  - Race badge destacado: "◈ {Race}" com cor da squad
  - Tagline (em itálico, quote style)
  - Squad Role (1-2 frases, white/80)
  - Model badge + Color badge
  - CTA "Instalar squad"

### SEÇÃO 2: Identidade & Especialização
- Esquerda: Abilities (cards ou lista destacada com ícone/bullet accent)
- Direita: Archetype (se existir) + lore da raça

### SEÇÃO 3: Autoridades Exclusivas (se existir)
- Manter seção atual mas com visual mais forte

### SEÇÃO 4: Tools (sidebar ou grid)
- Manter tools, mas em grid menor

### SEÇÃO 5: Siblings (manter igual)

## Acceptance Criteria
- [x] AC1: Hero exibe foto real do agente (`/agentes/{slug}.png`) via next/image — não mais initials
- [x] AC2: `tagline` do agente exibida no hero (estilo quote/italic)
- [x] AC3: `race` do agente exibida com destaque (badge colorida com cor da squad)
- [x] AC4: `squadRole` exibido como parágrafo de destaque abaixo do tagline

- [x] AC7: Layout responsive (mobile-first)
