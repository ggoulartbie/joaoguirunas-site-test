# Story F2.1: Configuração Supabase Auth (email/senha + Google OAuth)

## Objetivo

Configurar providers de autenticação no painel Supabase: email/senha (com confirmação por email) e Google OAuth, ambos integrados ao projeto.

## Acceptance Criteria

- [x] AC1: Provider Email habilitado com "Confirm email" ativo e templates customizáveis
- [x] AC2: Provider Google OAuth configurado com Client ID/Secret válidos e redirect URI correto (`{NEXT_PUBLIC_APP_URL}/auth/callback`)
- [x] AC3: SMTP custom configurado via Resend (ou Supabase nativo no MVP) — emails de confirmação chegam
- [x] AC4: Site URL e Redirect URLs configuradas para `localhost:3000`, `*.vercel.app` (preview) e domínio de produção (placeholder)
- [x] AC5: Trigger `handle_new_user` (criado em F1.2) testado: signup via email cria `profiles` automaticamente com role STUDENT — confirmado pelo sites-data
- [x] AC6: Documentação em `docs/smart-memory/agents/data-engineer/auth-config.md` listando configs do painel (não estão em código)

## Escopo

**IN:**
- Configuração de providers no painel Supabase
- Templates de email default (custom virá em F2.4)
- Documentação dos passos manuais

**OUT:**
- UI de login/cadastro (F2.2)
- Middleware (F2.3)
- Templates Resend customizados (F2.4)
