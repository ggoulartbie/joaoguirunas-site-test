# Story F7.7: /perfil — histórico, matrículas, recompra, renovação

## Objetivo

Página `/perfil` com edição de dados, histórico de pagamentos, lista de matrículas (ativas e expiradas), botão "Estender por R$ X" e toggle de renovação automática.

## Acceptance Criteria

- [ ] AC1: Edição de nome, bio, foto (upload pro bucket `avatars`); trocar senha
- [ ] AC2: Seção "Histórico de pagamentos": tabela com data, cohort, kind, valor, status, link pro recibo Stripe
- [ ] AC3: Seção "Minhas matrículas":
  - Tab "Ativas": cohort, prazo restante, % de progresso geral, toggle de renovação automática (se `cohorts.allows_auto_renewal=true`)
  - Tab "Expiradas": cohort, data de expiração, botão "Estender por R$ X" → leva ao checkout de extension (F7.3)
- [ ] AC4: Toggle de renovação automática chama Server Action que cria/cancela Stripe Subscription Schedule (F7.5)
- [ ] AC5: Botão "Estender" pré-aplica preço de extension (regra D11/D12 do ADR-001)
- [ ] AC6: Mobile responsive

## Escopo

**IN:**
- Página de perfil completa
- Botões/toggles que disparam actions de F7

**OUT:**
- Edição de email (Supabase precisa fluxo separado)
- Apagar conta (LGPD — futuro)

## Contexto Técnico

- Foto: bucket `avatars` (público), upload com Server Action
- Listagens: paginação se > 50 items
- Toggle de renovação dispara `enableAutoRenewal(membershipId)` ou `disableAutoRenewal(membershipId)`
