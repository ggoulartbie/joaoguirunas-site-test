# Story F9.15: Auditar APIs públicas e jobs (webhooks, crons, certificado)

## Objetivo
Auditar segurança das rotas API que NÃO estão sob `/api/admin/*`: `/api/webhooks/stripe`, `/api/cron/daily`, `/api/cron/hourly`, `/api/certificado/[code]`. Foco em verificação de assinatura/secret, idempotência, exposição pública e response shape.

## Acceptance Criteria
- [ ] AC1: `/api/webhooks/stripe`:
  - Verifica assinatura via `stripe.webhooks.constructEvent` com `STRIPE_WEBHOOK_SECRET`. Sem essa verificação → 401.
  - Idempotência implementada (event-id deduplicado) — testar enviando mesmo event 2x e confirmar que side-effect (criar matrícula, atualizar pagamento) só ocorre 1x.
  - Falhas internas retornam 500 sem stack trace; logam server-side.
  - Eventos `checkout.session.completed`, `invoice.paid`, `customer.subscription.deleted` cobertos sem vazar info sensível na response.
- [ ] AC2: `/api/cron/daily` e `/api/cron/hourly`:
  - Header `Authorization: Bearer ${CRON_SECRET}` exigido (ou validação Vercel cron). Sem header → 401.
  - Listar jobs executados em cada cron. Documentar em `docs/smart-memory/agents/research/cron-jobs-inventory.md`.
  - Jobs idempotentes (executar 2x não duplica side-effects).
- [ ] AC3: `/api/certificado/[code]`:
  - É público por design (página de verificação) — confirmar que NÃO retorna dados sensíveis (apenas: aluno, curso, data emissão, código). Não vazar email, telefone, CPF.
  - Code é validado como UUID/hash; tentar SQL injection / path traversal no `[code]` → bloqueado.
  - Rate limit considerado (mesmo que via Vercel/Cloudflare) — documentar se ausente.
- [ ] AC4: Nenhuma das 4 rotas vaza service role key, secrets, ou conteúdo de `.env` em mensagem de erro.
- [ ] AC5: Documento `docs/smart-memory/agents/research/api-public-audit.md` lista cada rota com: método, auth, payload, response, side-effects, idempotência.
- [ ] AC6: Bugs P0/P1 CORRIGIDOS na story; P2+ documentados.
