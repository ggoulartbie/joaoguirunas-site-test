# Story F2.2: Páginas de auth — login, cadastro, recuperar senha

## Objetivo

Criar as 4 páginas públicas de autenticação com UI alinhada ao KV do site, validação Zod + React Hook Form, e Server Actions que usam o client Supabase server-side.

## Acceptance Criteria

- [x] AC1: `app/(academy)/academy/login/page.tsx` com form (email, senha) + links pra cadastro e recuperar senha — open redirect protegido via `?next=` validado no client
- [x] AC2: `app/(academy)/academy/cadastro/page.tsx` com form (nome, email, senha, confirmar senha)
- [x] AC3: `app/(academy)/academy/recuperar-senha/page.tsx` com form (email) que dispara reset via Supabase
- [x] AC4: `app/(academy)/academy/redefinir-senha/page.tsx` que valida token e permite criar nova senha
- [x] AC5: `app/auth/callback/route.ts` Route Handler que troca código OAuth por sessão e redireciona para `/academy/dashboard`
- [x] AC6: Validação Zod em todos os forms; mensagens de erro em português
- [x] AC7: Após signup com email/senha, página informa "Confira seu email para confirmar"
- [x] AC8: Após login bem-sucedido, redireciona para `/academy/dashboard` (ou `?next=` se presente)
- [x] AC9: UI usa tokens KV (var(--ember), var(--void), Fraunces display, type-mono); logo Academy no header
- [x] AC10: Erros Supabase traduzidos — "Email ou senha incorretos", "Este email já está cadastrado"
- [x] AC11: Mobile responsive (layout flex col em mobile, split panel em lg+)

## Escopo

**IN:**
- 4 páginas + 1 callback OAuth
- Validação e i18n de erros
- UI alinhada ao KV

**OUT:**
- Middleware (F2.3)
- Templates de email Resend (F2.4)
- Onboarding wizard (Fase 8)
- Botão Google OAuth (disponível via callback, mas UI button não adicionado — fora do MVP imediato)
