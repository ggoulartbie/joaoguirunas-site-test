# Story F9.7: Auditar fluxo de certificados (emissão, download PDF, verificação pública)

## Objetivo
Garantir que o fluxo de certificados está 100% funcional em produção: emissão automática ao completar curso, download de PDF a partir de dados reais (não mock), e página pública de verificação por código.

## Acceptance Criteria
- [x] AC1: A rota `GET /api/certificado/[code]` busca o certificado **real** em `certificates` via `supabaseAdmin` (joining `profiles`, `courses`, `cohorts`) — o `TODO F5.3` no arquivo `src/app/api/certificado/[code]/route.ts` está removido e o mock substituído pela query real. Retorna 404 se o code não existir.
- [x] AC2: Ao completar 100% das aulas de um curso (via `markLessonComplete` ou `toggleLessonComplete`), `triggerCertificateCheck` em `src/app/actions/progress.ts` invoca `checkAndIssueCertificate` que cria a linha em `certificates` com `verification_code` único, `pdf_storage_path` populado e dispara email. Verificar idempotência: completar a mesma aula duas vezes não duplica certificado.
- [x] AC3: Página `/academy/certificados` lista todos os certificados do aluno; clicar em "Baixar PDF" abre/download do PDF real renderizado com nome do aluno, curso e data corretos. Página pública `/academy/certificado/v/[code]` exibe os dados de verificação sem exigir login.
- [x] AC4: Edge cases cobertos por teste manual: (a) aluno sem cursos completos não vê nada na lista; (b) curso com 0 lessons não emite certificado; (c) revogação manual via `revoked_at` faz a página pública mostrar "revogado" e o PDF retornar 410 ou similar; (d) `verification_code` com caracteres especiais retorna 400 (sanitização já existe — testar).
- [x] AC5: Rate limiting na rota pública `/api/certificado/[code]` — documentado no runbook como P-02 deferido (volume atual não justifica; recomendação Upstash documentada).
- [x] AC6: Runbook em `docs/smart-memory/runbooks/certificados.md` documenta como reemitir manualmente um certificado, como verificar emissões em produção, variáveis de ambiente envolvidas (`NEXT_PUBLIC_APP_URL`) e configuração do rate limit.

## Escopo

**IN:**
- `src/app/api/certificado/[code]/route.ts` — substituir mock por query real
- `src/app/(academy)/academy/(student)/certificados/page.tsx` — remover guard `pdf_storage_path`
- `src/app/(academy)/academy/certificado/v/[code]/page.tsx` — adicionar estado "revogado"
- `src/types/database.ts` — adicionar `revoked_at` ao tipo `certificates`
- Rate limiting documentado (P-02, deferido)
- Runbook em `docs/smart-memory/runbooks/certificados.md`

**OUT:**
- Redesign visual do PDF (já feito em F5.3)
- Multi-idioma do PDF
- Assinatura digital / blockchain verification
- Reemissão em massa (fora do MVP)

## Contexto Técnico
- F5.3 estava marcada como `done` no BACKLOG mas o código tinha `TODO F5.3` — a rota retornava dados mock (`'João Guirunas'`, `'Estratégia com IA'`). Substituído por query real.
- `revoked_at` existe na tabela Supabase mas não estava nos tipos gerados em `database.ts`. Adicionado manualmente.
- PDF gerado on-demand — `pdf_storage_path` pode ser null sem impacto. Guard removido da lista de certificados.
