# Story F1.6: Buckets de Storage + policies

## Objetivo

Criar os 4 buckets de Supabase Storage (`avatars`, `materials`, `certificates`, `covers`) com policies que respeitam a regra de acesso da plataforma.

## Acceptance Criteria

- [x] AC1: Bucket `avatars` criado como público com policy de upload pelo próprio usuário
- [x] AC2: Bucket `covers` criado como público com upload restrito a ADMIN
- [x] AC3: Bucket `materials` criado como privado; acesso via signed URL 5 minutos
- [x] AC4: Bucket `certificates` criado como privado; acesso via signed URL 1 hora
- [x] AC5: Policies de `materials` validam `has_access()` na lesson via path `materials/{lesson_id}/{material_id}.{ext}`
- [x] AC6: Policies de `certificates` validam `auth.uid() = user_id` via path `certificates/{user_id}/{certificate_id}.pdf`
- [x] AC7: Migration SQL versionada aplicada via `supabase db push`
- [x] AC8: Documentação em `docs/smart-memory/agents/data-engineer/storage.md`
