# Story F11.1: Migrar crons para Supabase pg_cron + Edge Functions

## Objetivo
Eliminar dependência do Vercel Pro para cron jobs, migrando toda a lógica agendada para Supabase (pg_cron + Edge Functions). Deploy fica 100% no plano Hobby.

## Acceptance Criteria

- [x] AC1: Lógica de expirar matrículas (`expire_memberships`) migrada para pg_cron — SQL puro, roda 1x/dia
- [x] AC2: Lógica de lembretes D-15/D-7/D-3 migrada para Edge Function chamada via pg_cron
- [x] AC3: Lógica de renovação automática migrada para Edge Function chamada via pg_cron
- [x] AC4: Lembrete "sessão em 1 hora" migrado para Edge Function chamada via pg_cron (schedule: `0 * * * *`)
- [x] AC5: `/api/cron/daily` e `/api/cron/hourly` em Next.js removidos ou mantidos como fallback desactivado
- [x] AC6: `vercel.json` sem nenhum cron job — zero dependência de plano Pro
- [ ] AC7: Edge Functions deployadas e activas no projecto Supabase `mksmmpfyqowuzjcchhkl`

## Notas
Pós-launch. Não bloqueante para go-live.

AC7 requer deploy via CLI: `supabase functions deploy cron-daily --project-ref mksmmpfyqowuzjcchhkl` e `supabase functions deploy cron-hourly --project-ref mksmmpfyqowuzjcchhkl`. Delegar ao sites-devops.

Após deploy, configurar `app.supabase_functions_url` e `app.service_role_key` no Supabase Dashboard → Settings → Vault, ou via SQL:
```sql
alter system set app.supabase_functions_url = 'https://mksmmpfyqowuzjcchhkl.supabase.co/functions/v1';
alter system set app.service_role_key = '<SERVICE_ROLE_KEY>';
select pg_reload_conf();
```
