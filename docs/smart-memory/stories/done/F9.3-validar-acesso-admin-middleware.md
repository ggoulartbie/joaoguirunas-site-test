# Story F9.3: Validar gate de acesso /academy/admin (middleware + RBAC)

## Objetivo
Garantir que o middleware Next.js trata corretamente os três cenários para qualquer rota sob `/academy/admin/*`: (a) anônimo → redireciona para `/academy/login?next=...`; (b) autenticado sem `role='ADMIN'` → redireciona para `/academy/403`; (c) autenticado com `role='ADMIN'` → renderiza a página.

## Acceptance Criteria
- [x] AC1: Acesso anônimo a `/academy/admin`, `/academy/admin/cursos`, `/academy/admin/turmas`, `/academy/admin/pagamentos`, `/academy/admin/usuarios`, `/academy/admin/forum`, `/academy/admin/moderacao` redireciona (HTTP 307/308) para `/academy/login` com query `next=` apontando para o caminho original.
- [x] AC2: Usuário com `role='STUDENT'` tentando acessar qualquer `/academy/admin/*` cai em `/academy/403`.
- [x] AC3: Usuário `joao@growthsales.ai` (já ADMIN) acessa `/academy/admin` e a página de dashboard renderiza sem 500 e sem hidratação quebrada.
- [x] AC4: As Server Actions e páginas dentro de `(admin)/admin/*/page.tsx` têm proteção via `requireAdmin()` centralizada no `layout.tsx`.
- [x] AC5: A rota `/academy/admin` existe e renderiza `admin/page.tsx`; matcher `/academy/admin/:path*` está correto no middleware.
- [x] AC6: Suite Playwright cobre os três cenários em `e2e/academy-admin-gate.spec.ts`.

## Resultado da Auditoria

### Middleware (src/middleware.ts)
- Linhas 82-92: gate ADMIN correto — verifica `profile.role !== 'ADMIN'` e redireciona para `/academy/403`.
- Matcher inclui `/academy/admin/:path*` — cobre todas as sub-rotas.
- Redirect para login com `?next=<pathname>` implementado corretamente (linha 76-78).

### Página /academy/403
- Existe em `src/app/(academy)/academy/403/page.tsx` — bem formada, com link de volta ao dashboard.

### Defesa em profundidade (layout.tsx)
- **Gap encontrada:** nenhuma page.tsx admin chamava `requireAdmin()`.
- **Fix aplicado:** adicionado `await requireAdmin()` no `src/app/(academy)/academy/(admin)/layout.tsx`, que protege todas as páginas filhas em um único ponto.
- Server Actions já tinham `requireAdmin()` em cada função — OK.

### Suite E2E
- `e2e/academy-admin-gate.spec.ts` já existia com cobertura completa dos 3 cenários.
