# Story F5.2: /agenda + live_sessions

## Objetivo

Página `/agenda` com encontros ao vivo das cohorts ativas do aluno, libera `meeting_url` server-side apenas 30min antes do horário, exporta `.ics`.

## Acceptance Criteria

- [x] AC1: `app/(student)/agenda/page.tsx` lista live_sessions futuras das cohorts onde o aluno está ACTIVE
- [x] AC2: Sessions passadas aparecem em seção separada "Encontros anteriores"; se têm `recording_url` mostram badge "Gravação disponível"
- [x] AC3: Card de session mostra: título, data/hora, duração, cohort de origem
- [x] AC4: `meeting_url` só exposto quando `now >= scheduled_at - 30min && now <= scheduled_at + 120min` — verificação server-side; `null` fora da janela
- [x] AC5: Antes de 30min: mostra horário disponível `linkAvailableTime(scheduledAt)` em vez do botão
- [x] AC6: Botão "Adicionar à agenda (.ics)" via `ICSDownloadButton` component
- [x] AC7: Recording: link aparece em sessions passadas se `recording_url` preenchido
- [x] AC8: Mobile responsive
- [x] AC9: Dashboard mostra próximo encontro (implementado em F3.5)
