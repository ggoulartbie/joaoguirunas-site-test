# Story F10.5: Verificar + completar webhook Stripe → cria conta automaticamente

## Objetivo
Garantir que quando um aluno compra o Curso Online via Stripe, o webhook cria automaticamente a conta Supabase, o perfil e a matrícula no cohort correcto, e envia o email de boas-vindas com acesso.

## Acceptance Criteria

- [ ] AC1: Evento `checkout.session.completed` no webhook — extrai `customer_email`, `customer_details.name`, e metadata `cohort_slug`
- [ ] AC2: Verifica se já existe user com esse email (`supabaseAdmin.auth.admin.listUsers()`) — se sim, só cria `cohort_member`; se não, cria user novo
- [ ] AC3: Criação de user: `supabaseAdmin.auth.admin.createUser()` com email, nome, `email_confirm: true`
- [ ] AC4: Cria `cohort_members` com status ACTIVE, `expires_at` calculado via `cohort.access_duration_days`
- [ ] AC5: Envia email "AccessGranted" via Resend com magic link para primeiro acesso (`generateLink()` type `magiclink`)
- [ ] AC6: `createCheckoutSession` passa `metadata: { cohort_slug: cohort.slug }` no session params (verificar se já está a fazer isso)
- [ ] AC7: Idempotência — se webhook for recebido duas vezes para o mesmo `checkout.session.id`, não cria user/member duplicado (usa `webhook_events` table)
- [ ] AC8: Testado com Stripe CLI: `stripe trigger checkout.session.completed` com metadata correcto

## Escopo
**IN:** webhook handler, createCheckoutSession metadata, email de boas-vindas
**OUT:** fluxo de extensão/renovação (já existe em F7.4/F7.5)
