# Story F9.18: P1-cleanup F9.12 — bugs B-02 a B-06

## Objetivo
Resolver os 5 bugs P1 abertos identificados pelo sites-qa durante o bug-hunt F9.12 da área do aluno: link Stripe `/test/` em produção, link `/turmas/` inexistente, cross-module navigation sem gate, e duas N+1 queries no fórum.

## Acceptance Criteria

- [x] **AC1 (B-02 — Stripe test link):** `src/app/(academy)/academy/(student)/perfil/PerfilClient.tsx:658` deve usar `https://dashboard.stripe.com/payments` em produção (não `/test/payments`). Solução: detectar ambiente via `process.env.NODE_ENV === 'production'` ou `NEXT_PUBLIC_STRIPE_MODE` e alternar a URL. Testar em dev (continua `/test/`) e em prod (vai para dashboard live).

- [x] **AC2 (B-03 — LockedContent → /turmas link):** `src/components/student/LockedContent.tsx:26` aponta para `/turmas/${cohortSlug}` que não existe como rota pública. Substituir por `/curso-online` com texto "Acesse nossos cursos disponíveis".

- [x] **AC3 (B-04 — Cross-module navigation sem gate):** `src/app/(academy)/academy/(student)/curso/[slug]/aula/[lesson-slug]/page.tsx:91-135` calcula `globalPrev`/`globalNext` a partir de `allModulesRaw` (todos os módulos do curso) sem filtrar por `accessibleModuleIds` da matrícula. Fix aplicado: resolve `accessibleModuleIds` via `cohort_members` + `cohort_courses` antes de calcular globalIndex. Edge case coberto: aluno na última aula liberada vê `globalNext = null`.

- [ ] **AC4 (B-05 — N+1 forum index):** `src/app/(academy)/academy/(student)/forum/page.tsx:48-57` faz uma query de `count` por categoria (N queries) e linhas 83–116 fazem 2 queries de count por thread (2*N queries). Fix: substituir por uma única query que retorne os counts agregados (ex: RPC SQL ou view). Sugestão: criar view `forum_categories_with_counts` e `forum_threads_with_counts` no Supabase, ou usar `select` com `forum_replies(count)` e `votes(count)` no mesmo query usando aggregate references do PostgREST. Critério: Network tab da página `/academy/forum` deve mostrar no máximo 3 queries Supabase (categories, threads, membership) — não N+1.

- [ ] **AC5 (B-06 — N+1 forum detail):** `src/app/(academy)/academy/(student)/forum/[category]/[slug]/page.tsx:114–120+` faz query de count de votos por reply (N queries). Fix: query agregada análoga ao AC4. Critério: página `/academy/forum/[category]/[slug]` com 20 replies deve fazer no máximo 4 queries Supabase (thread, replies, votes-thread, votes-replies-aggregate) — não 1+N.

- [ ] **AC6 (regressão):** Smoke manual após fix:
  - `/academy/perfil` (com pagamento) → link "Ver no Stripe" abre dashboard correto por ambiente
  - `/academy/curso/[slug]/aula/[lesson-slug]` em módulo bloqueado → link "Voltar para turmas" leva a destino válido (não 404)
  - Navegar pela última aula do último módulo liberado → botão "próxima" oculto ou desabilitado (não pula para módulo bloqueado)
  - `/academy/forum` carrega em <500ms server time com 20 threads
  - `/academy/forum/[cat]/[slug]` com 20+ replies carrega em <500ms server time

### Issues observacionais (não bloqueantes)

- **[CONCERN-1]** AC1 usa `process.env.NODE_ENV` em vez de `NEXT_PUBLIC_STRIPE_MODE` — funcionalmente equivalente para os ambientes atuais. Considerar variável dedicada se houver staging com chaves Stripe live no futuro. Não afeta segurança.
- **[CONCERN-2]** Aggregate count via PostgREST retorna array `[{ count: N }]` — precisa de cast `as unknown as { count: number }[]` (linhas 52, 95). Solução pragmática e idiomática para a tipagem atual do supabase-js. Considerar tipar via type generators no futuro.
- **[CONCERN-3]** Smoke manual <500ms server time descrito em AC6 não é verificado automaticamente — depende de monitoramento em prod. Aceito por: (a) typecheck limpo, (b) shape das queries comprovadamente reduz N+1 a constante, (c) não bloqueante por critério explícito do AC.
