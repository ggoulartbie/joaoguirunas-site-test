# Story F3.5: Dashboard + /meus-cursos + /curso/[slug]

## Objetivo

Páginas de entrada do aluno: dashboard com saudação e progresso, listagem de cursos liberados (união das cohorts ativas) e página de curso com módulos/aulas.

## Acceptance Criteria

- [ ] AC1: `app/(student)/dashboard/page.tsx` mostra:
  - Saudação + foto + nome
  - Card "Continue de onde parou" (última lesson com progresso > 0 e < 100%)
  - Próximo encontro ao vivo (se aplicável)
  - Cards das cohorts ativas com % de progresso geral (média de lessons completadas dos cursos liberados)
  - Aviso "Sua matrícula em X expira em N dias" se `expires_at` < 30 dias
- [ ] AC2: `app/(student)/meus-cursos/page.tsx` lista cursos da **união** das cohorts ativas; cada card mostra qual cohort libera; cursos parcialmente liberados indicam "Módulos X/Y disponíveis"
- [ ] AC3: `app/(student)/curso/[slug]/page.tsx` mostra módulos do curso; módulos liberados navegáveis; bloqueados com cadeado e CTA "Disponível na turma X"
- [ ] AC4: Lógica de "qual cohort libera": JOIN de `cohort_members` (active) com `cohort_courses`; respeita `included_module_ids`
- [ ] AC5: Estado de loading e skeletons em todas as 3 páginas
- [ ] AC6: Mobile responsive
- [ ] AC7: Performance: queries usam índices criados em F1.3 e F1.4

## Escopo

**IN:**
- 3 páginas + componentes auxiliares
- Lógica de cálculo de progresso e cohort-libera
- Skeletons e estados vazios

**OUT:**
- Página de aula (F3.3)
- /perfil (Fase 7 — junto com renovação)
- /agenda (Fase 5)

## Contexto Técnico

- Query "meus cursos" é a mais quente; considerar Server Component com cache de 30s
- Estado vazio: aluno sem cohort ativa vê CTA "Conhecer turmas" → `/turmas`
- Indicador de progresso: % de lessons completadas / total de lessons liberadas no curso
