## Notas técnicas dos fixes

### Fix 1 — `layoutId="card-glow"` causa animação errada

**Problema:** Framer Motion usa `layoutId` para animar transições compartilhadas entre componentes. Quando vários `ProfessionalCard` montam o mesmo `layoutId`, o glow "salta" visualmente entre o último e o novo selecionado, já que o motion trata como um único elemento compartilhado.

**Fix:** remover `layoutId` do `motion.div` de glow e usar estilo condicional:

```tsx
<motion.div
  className={cn(
    'absolute inset-0 rounded-lg pointer-events-none transition-all duration-300',
    active
      ? 'bg-[#FF4400]/5 shadow-[0_0_32px_rgba(255,68,0,0.15)]'
      : 'bg-transparent shadow-none',
  )}
/>
```

Cada card passa a ter seu próprio glow independente — múltiplos ativos coexistem sem conflito.

### Fix 2 — `aria-pressed` nos toggles

Botões de toggle (seleção múltipla) devem usar `aria-pressed` para comunicar estado. Código:

```tsx
<button
  type="button"
  onClick={onToggle}
  aria-pressed={active}
  aria-label={`${professional.label}, R$ ${professional.cost.toLocaleString('pt-BR')}/ano, ${active ? 'selecionado' : 'não selecionado'}`}
  className={cn(/* ... */)}
>
```
