# Story WK3-2.3: Slides 06-08 (números + clientes + processo interno)

## Objetivo
Provar tração — receita, base de clientes e como a Growth opera internamente com IA — em três slides sequenciais.

## Acceptance Criteria
- [ ] AC1: `src/app/workshop-3/numeros/page.tsx` (`<Workshop3DeckLayout slug="numeros">`) mostra timeline visual de 4 pontos: **0 → 150k/mês** (em 18m), **500k/mês** (projeção dez/2026), **R$ 6M** (meta 2027); cada número em escala display grande, com label pequeno em mono abaixo
- [ ] AC2: Slide números enfatiza o **0** inicial e o **6M** final usando cor accent Growth; saltos intermediários em foreground/70
- [ ] AC3: `src/app/workshop-3/clientes/page.tsx` (`<Workshop3DeckLayout slug="clientes">`) lista clientes em grid ou linha: **Sisprime**, **Blue3**, **Viva América**, **Argoplan**, com sufixo "+ outros" discreto
- [ ] AC4: Slide clientes usa apenas tipografia (sem logos externos — não há tempo de buscar e validar uso). Nomes em peso semibold/bold, separados por divisores sutis ou em grid
- [ ] AC5: `src/app/workshop-3/processo-interno/page.tsx` (`<Workshop3DeckLayout slug="processo-interno">`) com mensagem-âncora: "IA em todo processo interno" + linha de apoio "Menos recurso, mais entrega"
- [ ] AC6: Slide processo-interno inclui 3 mini-blocos ou bullets que ilustrem onde a IA opera (ex: "Operação · Comercial · Produto" ou "PMO · Prospecção · QA") — escolher trio que reforce a tese
- [ ] AC7: Cada página exporta `metadata` específico (`title` ex: "06 · Números — Growth Sales", `robots.noindex`)
- [ ] AC8: `export const dynamic = 'force-dynamic'` nas três
- [ ] AC9: Slide números tem hierarquia visual clara para o apresentador acompanhar o ritmo de fala (números em escala muito grande, labels pequenos)

## Escopo

**IN:**
- 3 páginas (`numeros`, `clientes`, `processo-interno`)
- Tipografia escalonada para números
- Lista de clientes em texto puro

**OUT:**
- Logos dos clientes (sem tempo de validar uso de marca)
- Gráfico/chart real (timeline é tipográfica, não SVG/canvas)
- Logo wall com imagens

## Contexto Técnico

Slide 06 é o "wow moment" da apresentação — números devem dominar visualmente a viewport. Considere `text-6xl md:text-8xl` ou similar para os valores.

Slide 07 funciona como prova social rápida — 30s no script. Sem floreio.

Slide 08 amarra "produtos + tração + processo" — prepara o terreno para a reflexão final do slide 09.
