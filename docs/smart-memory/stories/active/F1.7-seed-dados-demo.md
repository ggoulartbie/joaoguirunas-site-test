# Story F1.7: Seed de dados demo

## Objetivo

Popular o banco com dados realistas para desenvolvimento das próximas fases: 1 admin (João), 2 students teste, 1 mentor, 1 course demo, 2 cohorts (online + mentoria), cross extension Online→Mentoria, 3 forum_categories.

## Acceptance Criteria

- [ ] AC1: `supabase/seed.sql` versionado com inserts idempotentes (`on conflict do nothing`)
- [ ] AC2: 1 profile com `role='ADMIN'` (email/nome João Guirunas)
- [ ] AC3: 2 profiles com `role='STUDENT'` (aluno-online, aluno-mentoria)
- [ ] AC4: 1 profile com `role='MENTOR'`
- [ ] AC5: 1 course demo com slug `aiox-fundamentos`, 2 modules × 2 lessons cada (4 lessons total). Pelo menos 1 lesson de cada `kind`: VIDEO, READING. Pelo menos 1 com `content_format='MDX'` e 1 com `content_format='MARKDOWN'`
- [ ] AC6: Cohort `online-padrao`: `is_purchasable=true`, `has_public_page=true`, `entry_price_cents=49999`, `extension_price_cents=29999`, `access_duration_days=180`, libera todos os módulos do course demo
- [ ] AC7: Cohort `mentoria-maio-2026`: `is_purchasable=false` (matrícula manual), `entry_price_cents=870000`, `extension_price_cents=199900`, `access_duration_days=180`, `has_live_sessions=true`, `has_support=true`
- [ ] AC8: 1 `cohort_cross_extensions` com `source=online-padrao`, `target=mentoria-maio-2026`, `days_granted=180`, descrição "Mentorado que comprar Online ganha 180 dias adicionais na Mentoria"
- [ ] AC9: 3 `forum_categories`: "Dúvidas Técnicas", "Geral", "Networking"
- [ ] AC10: aluno-mentoria tem `cohort_members` ACTIVE em `mentoria-maio-2026` (matrícula manual demo)
- [ ] AC11: aluno-online tem `cohort_members` ACTIVE em `online-padrao`
- [ ] AC12: Comando `supabase db reset` repopula tudo limpo
- [ ] AC13: Função `has_access()` testada manualmente: aluno-online vê lessons; aluno-mentoria vê lessons; admin vê tudo; user inexistente recebe `false`

## Escopo

**IN:**
- Seed SQL idempotente
- Dados que cobrem os principais cenários: cohort vendável, cohort privada, cross extension, múltiplos roles
- Smoke test manual de `has_access()`

**OUT:**
- Vídeos reais (placeholder Vimeo ID)
- Imagens reais nos buckets (paths fictícios)
- Dados de pagamento (Fase 7)

- Seed roda **após** policies RLS estarem ativas → usar `supabase db reset` (executa como service_role)
- IDs hardcoded em UUIDs fixos para facilitar reset/replay (`'00000000-0000-0000-0000-000000000001'` etc)
- Hash de senha: usar Supabase Auth API ou criar profiles manualmente sem auth.users (admin pode criar via dashboard depois)
- Vimeo `video_id` placeholder até Fase 3 plugar Vimeo real
