# Story WK3-2.1: Slides 01-03 (abertura + pergunta + tese)

## Objetivo
Implementar os três primeiros slides do deck — abertura institucional, pergunta provocativa e tese central da Growth — usando o `Workshop3DeckLayout`.

## Acceptance Criteria
- [ ] AC1: `src/app/workshop-3/quem-somos/page.tsx` renderiza `<Workshop3DeckLayout slug="quem-somos">` com: nome **Growth Sales**, linha "18 meses no mercado", lista visual de 2 produtos (WorkOS + Orquestrador Comercial) com 1 linha de descrição cada
- [ ] AC2: `src/app/workshop-3/pergunta/page.tsx` renderiza `<Workshop3DeckLayout slug="pergunta">` com **um único bloco** de pergunta em fonte display grande: "Por onde começa um negócio com IA?" (sem corpo de texto, apenas a pergunta para gerar pausa cênica)
- [ ] AC3: `src/app/workshop-3/tese/page.tsx` renderiza `<Workshop3DeckLayout slug="tese">` com texto principal: "Negócio se desenvolve em cima de uma **necessidade** — não o contrário." + linha de apoio: "IA acelera a entrega disso."
- [ ] AC4: Em todos os 3 slides, ênfase visual em palavras-chave (negrito ou cor accent Growth) — `necessidade`, `IA`, `acelera`
- [ ] AC5: Cada página exporta `metadata` com `title` específico (ex: "01 · Quem somos — Growth Sales") e `robots: { index: false }`
- [ ] AC6: Cada página tem `export const dynamic = 'force-dynamic'`
- [ ] AC7: Sem componentes inline pesados — usar apenas JSX simples + tailwind/CSS inline já presente no padrão workshop-2
