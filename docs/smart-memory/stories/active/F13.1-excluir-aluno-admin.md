# Story F13.1: Excluir aluno no admin

## Objetivo
Permitir que o admin exclua permanentemente uma conta de aluno (auth.users + dados vinculados) a partir da modal `UserProfileModal` em `/academy/admin/usuarios`, com confirmação dupla e tratamento de erro.

## Acceptance Criteria

- [ ] **AC1** — Botão "Excluir conta" presente na modal `UserProfileModal`, separado visualmente do botão "Banir conta": estilo vermelho/destrutivo, ícone de lixeira (lucide-react `Trash2`), label claro. O botão aparece para qualquer usuário (banido ou não), exceto para o próprio admin logado (não pode se autoexcluir).
- [ ] **AC2** — Ao clicar em "Excluir conta", abre modal de confirmação com:
  - Texto de aviso explícito: "Esta ação é irreversível. Todos os dados vinculados à conta serão removidos permanentemente (perfil, progresso, comentários, certificados, notificações)."
  - Input de texto obrigatório onde o admin deve digitar exatamente `EXCLUIR` (case-sensitive) para habilitar o botão de confirmação.
  - Botão "Cancelar" (sempre habilitado) e botão "Excluir permanentemente" (habilitado apenas após digitação correta).
- [ ] **AC3** — Server action `deleteUser(userId: string)` em `src/app/(academy)/academy/(admin)/admin/usuarios/actions.ts`:
  - Valida sessão admin (RBAC) antes de executar.
  - Chama `supabaseAdmin.auth.admin.deleteUser(userId)` — cascateia via FK ON DELETE em `profiles`, `lesson_progress`, `comments`, `certificates`, `notifications`, `cohort_members`.
  - Mantém registros em `payments` (histórico financeiro): `payments.user_id` deve permitir NULL ou ter ON DELETE SET NULL. Se a FK atual for ON DELETE CASCADE/RESTRICT, ajustar via migration acompanhante.
  - Retorna `{ success: true }` ou `{ error: string }` (não lança).
- [ ] **AC4** — Após sucesso: modal fecha, usuário desaparece da tabela em `UsersClient.tsx` (via `router.refresh()`), toast verde "Conta excluída com sucesso".
- [ ] **AC5** — Erro tratado: se `deleteUser` retornar erro do Supabase ou outra exception, exibe toast vermelho com mensagem amigável ("Não foi possível excluir a conta. Tente novamente."), sem crashar a UI. Erro logado no console e (se configurado) enviado ao Sentry.

## Escopo

**IN:**
- Botão "Excluir conta" na `UserProfileModal`.
- Modal de confirmação com input de digitação `EXCLUIR`.
- Server action `deleteUser(userId)` com validação de admin e chamada a `auth.admin.deleteUser`.
- Migration acompanhante (se necessária) para ajustar FK de `payments.user_id` para `ON DELETE SET NULL`.
- Refresh da tabela de usuários após delete.
- Tratamento de erro com toast.
- Guard para impedir admin de se autoexcluir.

**OUT:**
- Exclusão em lote (multi-select).
- Export/backup dos dados antes do delete (LGPD compliance fica para story separada se necessário).
- Página dedicada de "lixeira" / soft delete recuperável.
- Notificação por email ao usuário excluído.
- Audit log persistente (se for necessário, criar story separada).

## Contexto Técnico

**Arquivos afetados:**
- `src/app/(academy)/academy/(admin)/admin/usuarios/actions.ts` — adicionar `deleteUser` action
- `src/app/(academy)/academy/(admin)/admin/usuarios/UsersClient.tsx` — wiring do botão e modal de confirmação no `UserProfileModal`
- `supabase/migrations/{timestamp}_payments_user_id_set_null_on_delete.sql` — se necessário ajustar FK

**Dependências:**
- `supabaseAdmin` já configurado (usado em ban/unban existentes).
- RBAC admin já validado pelo middleware/`(admin)` route group.
- Cascade em FKs de `profiles`, `cohort_members`, `lesson_progress`, `comments`, `certificates`, `notifications` precisa ser verificado — se alguma estiver RESTRICT, ajustar via migration.

**Constraints:**
- `auth.admin.deleteUser` é hard delete em `auth.users` — não há rollback.
- Idempotência: se chamado novamente após delete bem-sucedido, retornar `{ success: true }` (usuário já não existe).
- Não permitir delete do próprio admin logado (`session.user.id !== userId`).

**Decisões:**
- Confirmação dupla via digitação de `EXCLUIR` (não checkbox simples) — risco alto, ação irreversível.
- `payments` mantidos por compliance fiscal (histórico de receita).
- Cascade em `cohort_members` é aceitável (são dados acessórios, não financeiros).
