# Story F1.4: Migration — Pagamento + Progresso + Comunidade + Auxiliares

## Objetivo

Completar o schema com as tabelas restantes: pagamentos, cupons, webhook_events, progresso, certificados, comments, fórum (categories/threads/replies/votes) e notifications.

## Acceptance Criteria

- [ ] AC1: `payments` com `purchase_kind` (ENTRY/EXTENSION/AUTO_RENEWAL), `status` (PENDING/APPROVED/DECLINED/REFUNDED), Stripe IDs, FK opcional para `cohort_members(id)` (membership_id), `stripe_checkout_session_id` UNIQUE
- [ ] AC2: `coupons` por cohort, com `discount_kind` (PERCENT/FIXED), `applies_to` (ENTRY/EXTENSION/BOTH), unique (cohort_id, code), validade temporal
- [ ] AC3: `webhook_events` com `stripe_event_id` UNIQUE (idempotência), payload JSONB, success boolean, error_message
- [ ] AC4: `lesson_progress` com unique (user_id, lesson_id), `seconds_watched`, `completed`, `completed_at`
- [ ] AC5: `certificates` com `verification_code` UNIQUE (formato sugerido `AIOX-2026-X7K9P2`), FKs para user/course/cohort/membership, `pdf_storage_path`
- [ ] AC6: `comments` com self-reference `parent_comment_id` (threads de 1 nível), soft delete
- [ ] AC7: `forum_categories`, `forum_threads` (slug UNIQUE), `forum_replies` (self-reference), `votes` (CHECK que `thread_id` ou `reply_id` é NOT NULL; UNIQUEs separados)
- [ ] AC8: `notifications` com `notification_type`, `read_at`
- [ ] AC9: Índices em queries quentes: `payments(user_id, status)`, `lesson_progress(user_id, completed)`, `forum_threads(category_id, last_activity_at DESC)`, `notifications(user_id, read_at)`
- [ ] AC10: Tipos TS regenerados; `supabase db diff` vazio

## Escopo

**IN:**
- 11 tabelas: payments, coupons, webhook_events, lesson_progress, certificates, comments, forum_categories, forum_threads, forum_replies, votes, notifications
- Índices de performance para queries previstas

**OUT:**
- Função `has_access()` (F1.5)
- RLS (F1.5)
- Lógica de aplicação de cross extensions (Fase 7)

## Contexto Técnico

- `payments.amount_cents` em centavos
- `coupons.discount_value`: percentual 0-100 OU centavos (dependendo de `discount_kind`)
- `webhook_events` é o **único mecanismo de idempotência** Stripe — `UNIQUE` no `stripe_event_id` é load-bearing
- `votes.check (thread_id is not null or reply_id is not null)` — voto ou em thread, ou em reply
- `comments.parent_comment_id` self-reference; aplicação limita profundidade a 1
- ON DELETE CASCADE: lesson_progress→profiles e →lessons; comments→lessons; forum_replies→forum_threads; notifications→profiles
